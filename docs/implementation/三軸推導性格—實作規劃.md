# 三軸推導性格 — 實作規劃

> 對齊：人物屬性彙整 §二（SoulSeed → 三軸 → 體敏氣／本源）、討論 002（決策引擎／性格影響行為）

---

## 目標

由**同一個 soul_seed** 已展開的三軸（Amplitude, Frequency, Phase）**不再抽新 RNG**，推導出一組「性格」數值，供 NPC 決策權重、對話語氣、求職偏好使用。與星盤／體敏氣／本源同源，不新增儲存。

---

## 一、資料與 API

### 1.1 性格結構體

在 `db/entity.go`（或新檔 `db/personality.go`）定義：

```go
// Personality 由三軸推導的性格維度，皆 [0,1]。供決策／對話權重使用。
type Personality struct {
    Boldness    float64 // 強勢度／敢衝 ← Amplitude
    Sensitivity float64 // 敏感度／反應 ← Frequency
    Orderliness float64 // 秩序感／守規 ← Phase
}
```

語義對照：

| 軸 | 區間 | 性格維度 | 高／低解讀 |
|----|------|----------|------------|
| Amplitude | [0.1, 3.0] | Boldness | 高＝較敢冒險／議價／換工作；低＝保守忍讓 |
| Frequency | [0.5, 2.0] | Sensitivity | 高＝較易被情境影響；低＝遲鈍穩定 |
| Phase | [-1.0, 1.0] | Orderliness | 高＝守規矩、偏穩定工作；低＝隨性、偏自由業／行腳 |

### 1.2 展開函式

新增 **`ExpandSoulSeedToPersonality(seed int64) Personality`**：

- 與現有 `ExpandSoulSeedToBaseStats`、`ExpandSoulSeedToOriginSentence` **同一 RNG 序**：`rand.NewSource(seed)` 後取前 3 次 `Float64()`，得到 u1,u2,u3 → 算出 amp, freq, phase（常數 ampMin/Max 等與 entity.go 一致）。
- 正規化到 [0,1]：
  - `Boldness   = (amp - ampMin) / (ampMax - ampMin)`
  - `Sensitivity= (freq - freqMin) / (freqMax - freqMin)`
  - `Orderliness= (phase - phaseMin) / (phaseMax - phaseMin)`
- 回傳 `Personality{Boldness, Sensitivity, Orderliness}`。

**不寫入 DB**：需要時由 `entities.soul_seed` 即時計算即可；若日後效能有需求再考慮創角時快取到 entity 表或專用表。

## 二、實作步驟

### Phase 1：函式與型別（最小改動）

| 步驟 | 內容 |
|------|------|
| 1 | 在 `db/entity.go` 新增 `Personality` 結構體與 `ExpandSoulSeedToPersonality(seed int64) Personality`。 |
| 2 | 實作內沿用既有 `ampMin/Max`、`freqMin/Max`、`phaseMin/Max`，保證與三軸一致。 |
| 3 | （可選）在 `cmd/soulseed_demo` 或單元測試中呼叫一次，印出 Personality，確認與同 seed 的本源語感一致。 |

不改動現有 `ExpandSoulSeedToBaseStats`、`ExpandSoulSeedToOriginSentence`，避免影響既有邏輯。

### Phase 2：決策引擎使用（討論 002 決策引擎實作時）

| 步驟 | 內容 |
|------|------|
| 1 | NPC 決策入口（例如 `Think(ctx, entityID) Action`）取得該實體 `soul_seed`。 |
| 2 | 呼叫 `ExpandSoulSeedToPersonality(seed)` 得到 `Personality`。 |
| 3 | 依需求使用：例如「是否離職」時 Orderliness 高→留任權重提高；「選工作類型」時 Boldness 高→傾向高風險高報酬。 |

### Phase 3：對話／模板權重（Talk 串接對話模板時）

| 步驟 | 內容 |
|------|------|
| 1 | 取得對象 NPC 的 `soul_seed` → `ExpandSoulSeedToPersonality(seed)`。 |
| 2 | 同一 key 多句時，依 Boldness／Sensitivity 加權抽選（例如高 Boldness 偏向較衝的句子池或權重）。 |

## 四、驗收

- 同一 `soul_seed` 呼叫 `ExpandSoulSeedToPersonality` 與 `ExpandSoulSeedToOriginSentence`，語義一致（例如 seed 對應「霸道／敏銳／秩序」時，Boldness、Sensitivity、Orderliness 皆偏高）。
- 未新增 DB 欄位與 migration；依賴僅 `soul_seed`。
- Phase 1 完成即可合併；Phase 2／3 隨決策引擎與對話模板實作再接上。
