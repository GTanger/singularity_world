# 討論紀錄 002：NPC 需求驅動與求職機制

> 日期：2026-02-12  
> 前提：延續討論 001「身份與職業分離」；角色生成時無身份綁定，職業由場所與指派而來。

---

## 共識意向（收斂整理）

**需求驅動行為**：無身份 NPC 不會「天生就是店員」，要去當店員是因為**需要**。為所有角色（NPC＝玩家）創造需求（生理、物質→賺錢）→ 驅動求職與其他謀生行為。

**求職機制**：場所主人有職缺（如 max_staff、assignment 未滿），無職且鎂低的 NPC 有求職意圖；供需匹配（路過或撮合 tick）→ 寫 assignment → 獲得職業與身份。離職、解僱、流動自然發生。

**僧多粥少**：商舖滿招後，其餘 NPC 仍有謀生路——狩獵、採集、製作、**行腳商**（無場所綁定、自己即移動商舖）、乞討、賣藝、打零工。皆不需 assignment；身份從行為被觀測而來，與討論 001 一致。

**像真人玩家**：同構（同一套狀態／插座／世界）＋同驅（同一套需求驅動）＋同感（決策有偏好與脈絡）。真正差距在**決策**與**意圖**；「能有多像」取決於**決策引擎**：最簡＝固定優先序（鎂低→求職→賣貨→採集…），進階＝soul_seed 權重與目標堆疊，完整＝規劃／記憶／情境或 LLM。一句話：讓 NPC 像真人 ＝ 同一套世界 ＋ 同一套需求 ＋ 一套「會選」的決策邏輯。

## 使用者提問（記錄）

假設地圖有三十間商舖，已生成三十個 NPC 並綁定為商舖主人。接下來生成的無數 NPC 都沒有職業與身份——

**他們與商舖主人之間，要如何做出「求職→入職→獲取職業與身份」的行為？**

使用者自行聯想到**馬斯洛需求理論**：必須為所有角色（NPC＝玩家）創造**需求**——生理需求、物質需求、虛榮需求 → 產生**賺錢的需求** → 驅動求職行為。

核心問題：無身份的 NPC 不會「天生就是店員」，那他為什麼要去當店員？答案是**因為他需要**。

---

## 回覆：需求驅動行為（2026-02-12）

### 1. NPC 為什麼要工作？因為他會餓、會窮、會死

NPC 與玩家一致：有**鎂（貨幣）餘額**，會隨時間或消費（食物、住宿、裝備損耗）遞減。沒錢 → 狀態下降 → 有動機賺錢。不需複雜 AI，只需：NPC 的鎂低於閾值 → 觸發「求職意圖」。

### 2. 求職怎麼發生？供需匹配

- **場所主人**有職缺：如商舖需要 1~2 名店員，目前 assignment 不足 → 有空缺。
- **無職 NPC**有求職意圖：鎂低於閾值、或無 assignment。
- **匹配**：NPC 路過或附近 → 系統檢查「有缺嗎」→ 有缺 → 寫 assignment → 獲得職業與身份。
- 最低限度：場所有 max_staff 欄位、assignment 數量未滿、NPC 符合條件 → 自動指派。

### 3. 需求層級（第一版兩層即可）

| 層 | 需求 | 驅動行為 |
|---|------|---------|
| 生存 | 鎂 > 0、定期消耗 | 鎂不足 → 求職、接任務、乞討 |
| 安定 | 有固定收入（有 assignment） | 有職 → 留任；失職 → 再求職 |

後續可疊加：社交（偏好人多場所）、成就（攢錢開店）、自我實現（依 seed 性格偏好特定職類）。

### 4. 與插頭插座的對接

場所可暴露 Hire 插座；或更簡單：後端每隔一段時間跑「求職撮合 tick」——掃無職且鎂低 NPC、掃有空缺場所、距離／偏好匹配後自動寫 assignment。

### 5. 同時解決離職與流動

更好機會（更高薪、距離更近、性格偏好）→ 離職（刪舊 assignment、寫新的）；場所主人可解僱（刪除 assignment）。NPC 在商舖間自然流動。

## 延伸：僧多粥少——商舖滿招後，其餘 NPC 怎麼辦？（2026-02-12）

**使用者提問（記錄）**：如果商舖都滿招，其餘 NPC 該怎麼辦？能否像玩家一樣滿世界狩獵、採集、取材賺錢？成為沒有商舖的行腳商？還是…？

### 回覆

商舖滿招 ≠ 世界沒有謀生手段。職業（assignment）是「被場所主人指派」的一種身份，但**賺錢的方式不止被僱用這一條路**。無職 NPC 可以做的事：

#### A. 自給自足型（不依賴場所指派）

| 行為 | 說明 | 與系統對接 |
|------|------|----------|
| **狩獵** | 到野外房間擊殺怪物 → 掉落材料或鎂 | Attack 插座 + 戰鬥系統 + 掉落 |
| **採集** | 到有 `outdoor`/`garden`/`wilderness` tag 的房間 → 採集資源 | 可新增 Gather 插座 → 物品入背包 |
| **製作** | 有材料 → 在有 `forge`/`workshop` tag 的房間製作 → 成品可賣 | Craft 插座 |
| **行腳商** | 在 A 地買、B 地賣，賺差價 | Trade 插座 + 各房間/NPC 的買賣價差 |

這些行為**不需要 assignment**、不需要場所指派：NPC 靠自身行動（移動 + 插座動作）就能完成。

#### B. 非正式經濟型

| 行為 | 說明 |
|------|------|
| **乞討** | 在 gate/social 房間 → Beg 插座 → 從路人或系統獲得少量鎂 |
| **偷竊** | 高風險高回報（若系統支持） |
| **賣藝** | 在 social/tavern 房間表演 → 路人打賞 |
| **打零工** | 場所非正式需求：搬貨、送信，一次性任務而非長期 assignment |

#### C. 行腳商（特別展開）

行腳商就是**沒有場所綁定的商人**——他不被任何商舖僱用，也不擁有商舖，但他 **自己就是移動的商舖**：

- 背包裡有貨物（Inventory）
- 移動模式為 **pathfind** 或 **route**，在城鎮間巡迴
- 對其他角色暴露 **Trade 插座**（任時任地，因為他的「場所」就是他自己）
- 利潤來自：在 A 鎮以低價買 → 移動到 B 鎮以高價賣

這不需要 assignment，不需要場所指派——Trade 插座可以設為**預設 Agent 插座**（目前已有），或由 NPC 自身的行為模式決定。行腳商的身份不是被場所給的，而是**外人觀測到「這個人在賣東西」**→ 認知為商人。

#### D. 需求驅動的行為選擇邏輯

NPC 不需要「知道自己是行腳商還是獵人」，他只需要一組**決策優先序**：

```
1. 鎂低於閾值？
   ├→ 附近有空缺場所？ → 求職（寫 assignment）
   ├→ 背包有貨物？ → 找買家賣掉（Trade）
   ├→ 附近有可採集資源？ → 採集（Gather）
   ├→ 附近有可狩獵目標？ → 狩獵（Attack）
   └→ 以上都沒有 → 移動到其他區域重試 / 乞討
2. 鎂充足、無職 → 閒逛（pathfind）、社交
3. 鎂充足、有職 → 正常上班
```

**身份永遠是結果，不是輸入。** 他做了採集 → 別人看到他在採集 → 認知他是採集者。他做了買賣 → 別人認知他是行腳商。他什麼都沒做 → 路人。

#### E. 對系統的影響

- **經濟循環**：無職 NPC 從野外帶資源進城 → 賣給商舖或玩家 → 商舖轉手賣出 → 形成供應鏈。若只有被僱用才能賺錢，經濟會停滯。
- **世界觀活力**：有人在城裡工作、有人在野外冒險、有人趕路做生意——和真實世界一致，而非每個 NPC 都站在店裡。
- **與討論 001 一致**：身份從行為來，不從標籤來。無職不代表無事可做。

## 核心收斂：如何讓 NPC 像真人玩家？能有多像？（2026-02-12）

**使用者**：現在所作的一切都還沒脫離模板角色，濃縮所有問題後只剩一個核心——**如何讓 NPC 像真人玩家？能有多像？**

### 回覆摘要

**像真人玩家的維度**：狀態（同構）、動作（同插座）、身份（由行為與情境認知）、需求（要錢要活）、**決策**（自己選要做什麼）、視野、記憶、意圖。真正差距在**決策**與**意圖**——玩家會「自己想」要做什麼，NPC 多半是「被規則決定」。

**能有多像？三層**：

| 層級 | 內涵 |
|------|------|
| **同構** | 同一套狀態、插座、世界規則；玩家能做的結構上 NPC 也能做。 |
| **同驅** | NPC 也有需求（生存、賺錢、安定），需求驅動行為，不是腳本。 |
| **同感** | 決策有偏好、有脈絡、有一致性；soul_seed 推性格、目標堆疊、情境選擇；需要**決策引擎**。 |

**決策引擎**才是「能有多像」的瓶頸：最簡版＝固定優先序（鎂低→求職→賣貨→採集…）；進階＝soul_seed 權重、目標堆疊；完整＝目標規劃、簡單記憶、情境推理或每 NPC 一 LLM。

**一句話**：讓 NPC 像真人玩家 ＝ 同一套世界 ＋ 同一套需求 ＋ 一套「會選」的決策邏輯。
