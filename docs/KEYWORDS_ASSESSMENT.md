# 關鍵字完善度評估（組員端）

主管問：關鍵字所架構的夠完善了？組員為**大數據庫**：主管未提或僅一句者，由組員依概念補出**多項大數據建議**供主管決斷，而非反過來請主管補填。以下為評估與各區塊的**三至五種以上方案**。

---

## 已足夠支撐第一版的方向

- **設計核心、緣起、用語**：清楚，可直接作為共識。
- **視角與畫面、移動與操作**：足以定「第一版長什麼樣、怎麼操作」。
- **世界、觀測與坍縮**：概念完整，實作時再細化「觀測／接觸」的定義即可。
- **角色與物件、玩家＝NPC**：身份與規則一致，利於後端同一套邏輯。
- **架構與數據、社會系統、玩法類型、戰鬥**：有明確邊界，可依此收斂 MVP。

**結論**：關鍵字已夠用於推動下一步；以下區塊由組員提供**多項方案**，主管擇一或混合決斷即可。

---

## 偏薄或未定區塊 — 組員端大數據建議（每項 3～5+ 方案 ＋ 組員推薦與原因）

主管僅有概念（如「真實經濟」）或未提時，由組員列舉可行方案與參考；每區塊並附**組員推薦**（單選或複數融合）及**原因**，供主管決斷時參考。

---

### 1. 經濟（「真實經濟」的具體型態）

| 方案 | 簡述 | 參考／利弊 |
| --- | --- | --- |
| **A. 單一通用貨幣＋後端產消閉環** | 一種錢，由 NPC 產出（勞動、販售）與消耗（購物、稅、報酬），玩家參與同一循環；無系統「發錢」、無無中生有。 | 類 EVE／Albion 部分設計；利：易對齊「永序守衡」；弊：需嚴謹產消表。 |
| **B. 多幣／多資源（金／銀／糧／材）** | 多種可交易資源，各有產出與消耗管道，彼此可兌換或僅部分互通。 | 類 MUD 多資源、部分沙盒；利：社會分工明顯；弊：平衡複雜。 |
| **C. 以物易物為主、貨幣為輔** | 交易以物易物為常態，貨幣僅用於找零或特定 NPC 結算。 | 利：強「真實」感、減少通膨單一變數；弊：定價與介面較繁。 |
| **D. 全由 NPC 驅動物價（玩家為參與者）** | 物價、供需全由 NPC 行為與後端模擬決定，玩家買賣影響局部但不主導。 | 類單機經濟模擬；利：世界感強；弊：玩家可能難「致富」或需接受波動。 |
| **E. 時間綁定產出（每日／每季有限產量）** | 資源與錢有時間維度的產出上限，避免無限刷、通膨爆炸。 | 類農耕／季節經濟；利：與「時間」系統自然結合；弊：需設計時間流速與產出曲線。 |

**組員推薦**：**A＋E**（單一貨幣產消閉環 ＋ 時間綁定產出）。**原因**：A 對齊「不可無中生有」「永序守衡」、錢從 NPC 產消閉環來；E 防無限刷與通膨、且與「時間」區塊自然銜接，後端易做唯一真理。

主管決斷：可選 A～E 之一，或混合（例：A＋E）。

### 2. 死亡／懲罰／重生

| 方案 | 簡述 | 參考／利弊 |
| --- | --- | --- |
| **A. 無永久死亡，僅狀態懲罰** | 死亡後重生於定點，扣部分金錢／耐久／經驗或短暫 debuff；裝備不消失。 | 類多數 MMO；利：不勸退、與「下線變 NPC」易並存；弊：死亡重量感低。 |
| **B. 死亡即「離線化」一段時間** | 死亡後角色變成 NPC 一段時間（由後端驅動），玩家無法操作直到「復活」條件滿足。 | 與「玩家＝NPC」高度一致；利：敘事可融入；弊：需定義復活條件與時間。 |
| **C. 永久死亡（permadeath）** | 角色死亡即刪除或轉為純 NPC，玩家需新角色。 | 類 Roguelike／部分硬核 MUD；利：張力極高；弊：與 1～10 人、輕操作可能衝突。 |
| **D. 死亡掉裝備／背包部分** | 重生保留角色與等級，但失去身上或背包部分物品（可被撿或消失）。 | 類 Dark Souls／部分生存；利：有風險感；弊：需防 grief、需與經濟平衡。 |
| **E. 死亡僅影響「當前任務／狀態」** | 不扣裝備與錢，但當前進行中的任務或劇情分支重置或失敗。 | 利：輕懲罰、重敘事；弊：戰鬥若全自動，死亡觸發頻率需設計。 |

**組員推薦**：**A＋部分 B**（無永久死亡、狀態懲罰為主；可選「死亡後短暫離線化」）。**原因**：A 不勸退、與「下線變 NPC」並存、實作簡單；若希望死亡有敘事重量，可疊加 B（死亡後角色由後端驅動一段時間再復活），與「玩家＝NPC」一致且不需 permadeath。

主管決斷：准 推薦。

### 3. 時間

| 方案 | 簡述 | 參考／利弊 |
| --- | --- | --- |
| **A. 世界時間 1:1 現實** | 遊戲內時間流速與現實一致，有晝夜（可選）。 | 利：簡單、與「永遠在線」自然合；弊：事件節奏慢。 |
| **B. 世界時間加速（如 1 現實分＝1 遊戲日）** | 加快流速，有晝夜／季節，NPC 與經濟依遊戲日推進。 | 類動物森友會／部分經營；利：變化多；弊：需定義「一天」內產出與事件。 |
| **C. 無晝夜，僅事件時間軸** | 無視覺晝夜，但後端有「事件時間戳」與先後順序，用於任務、經濟、故事線。 | 利：實現簡單、重文字敘述；弊：無時間氛圍。 |
| **D. 晝夜／季節影響產出與事件** | 有晝夜與季節，影響採集、NPC 行為、部分事件觸發。 | 類 Stardew／部分 MMO；利：世界感強；弊：平衡與內容量需求大。 |
| **E. 僅「冷卻／CD」型時間** | 不做全球時間，只做技能／行動冷卻、下線任務倒數等。 | 利：最輕量；弊：與「真實經濟」「季節」類設計較難對齊。 |

**組員推薦**：**B＋C**（世界時間加速、有晝夜／季節 ＋ 後端事件時間軸）。**原因**：B 讓經濟與 NPC 故事有「日／季」節奏、與經濟區塊 E 銜接；C 保證後端有明確事件序，即使前期不做視覺晝夜也能用文字敘述「時序」，符合「重文字」；兩者並存可先做 C 再漸進加 B 的視覺或敘事表現。

主管決斷：准 推薦。

### 4. 同服玩家互動（1～10 人）

| 方案 | 簡述 | 參考／利弊 |
| --- | --- | --- |
| **A. 僅共處一世界，無直接交易／組隊** | 看得見彼此、同地圖，但無交易介面、無組隊；互動僅透過「同一個世界內的間接影響」。 | 利：實現簡單、糾紛少；弊：多人感弱。 |
| **B. 可交易、不可 PK** | 玩家間可交易物品／金錢，無直接對戰。 | 類部分 MMO 經濟服；利：真實經濟可延伸；弊：需防洗錢／外掛經濟。 |
| **C. 可交易＋可合意對戰** | 交易＋雙方同意才開戰（決鬥）。 | 利：有張力、可競技；弊：需規則防 grief。 |
| **D. 可組隊共鬥 NPC／事件** | 無 PvP，但可組隊打怪、解事件、共享部分獎勵。 | 類共鬥手遊；利：社交與難度可調；弊：需分配與掉落規則。 |
| **E. 全開放（交易＋組隊＋可 PK）** | 同服內全開放互動，死亡與經濟規則同上。 | 類硬核 MMO／部分 MUD；利：自由度高；弊：1～10 人易形成霸凌，需設計保護或懲罰。 |

**組員推薦**：**B**，或 **B＋D**（可交易、不可 PK；若要做共鬥則加 D）。**原因**：同服僅 1～10 人，全開放 PvP（E）易造成霸凌與流失；B 延伸「真實經濟」、實現簡單、糾紛少；若希望多人有「一起做一件事」的體驗，再加 D（組隊共鬥 NPC／事件），不引入 PK 即可兼顧社交與穩定性。

主管決斷：B+C+D。

### 5. 物件範圍與互動粒度

| 方案 | 簡述 | 參考／利弊 |
| --- | --- | --- |
| **A. 格點制（tile-based）** | 地圖為格點，角色與固體皆佔格；「不可穿越」= 該格有固體即不可進入。 | 類傳統 MUD／Roguelike；利：規則清晰、易做路徑與觀測；弊：視覺較方塊。 |
| **B. 格點＋多格佔位** | 大地圖仍格點，但大物件佔多格（如建築 2x2、3x3）。 | 利：兼顧規則與規模感；弊：碰撞與路徑稍複雜。 |
| **C. 向量／形狀碰撞** | 不用格點，用圓形或多邊形做碰撞；「不可穿越」由形狀判定。 | 利：視覺可更自由；弊：與「字符示意地形」需對齊。 |
| **D. 物件類別明訂** | 明訂：門、容器、採集點、建築、地形塊、可撿物等；每類有統一互動介面（點擊→後端回傳可做動作）。 | 利：全物件可互動可實作；弊：需維護類別表。 |
| **E. 混合：格點地形＋物件實體** | 地形用格點（可行／不可行），其上再疊「物件實體」（門、NPC、可撿物）各有互動。 | 類多數 2D 地城；利：清晰且可擴；弊：需分層資料結構。 |

**組員推薦**：**A＋D＋E**（格點制 ＋ 物件類別明訂 ＋ 格點地形＋物件實體混合）。**原因**：A 讓「不可穿越」與路徑、觀測坍縮的「視線」有清晰定義，與字符地形易對齊；D 讓「全物件可互動」有實作清單；E 是 A 與 D 的具體組合：地形層格點、其上實體分類互動，後端唯一真理易建模、前端顯示可漸進。

主管決斷：C+D。

### 6. 自創技能邊界（「符合世界規則」）

| 方案 | 簡述 | 參考／利弊 |
| --- | --- | --- |
| **A. 組合既有模組** | 技能由「效果模組」組合（傷害類型、範圍、條件、消耗），玩家選組合、不新增底層規則。 | 類卡牌構築／技能樹；利：不會無中生有、易平衡；弊：創意有上限。 |
| **B. 配方／藍圖制** | 「自創」= 符合既有配方或藍圖的解鎖與組合，新技能需透過遊戲內發現或 NPC 傳授。 | 利：全在世界內生成；弊：需大量配方設計。 |
| **C. 玩家提案、後端驗證** | 玩家可提交技能描述或參數，後端檢查是否在「允許參數空間」內，通過才生效。 | 利：有真·自創感；弊：需嚴謹參數空間與防濫用。 |
| **D. 僅 NPC／世界內事件可「創」技能** | 玩家不直接創技能，而是透過任務、門派、事件取得「新」技能（實為設計好的擴充）。 | 利：敘事與規則完全可控；弊：自創感較弱。 |
| **E. 技能即工具熟練度的延伸** | 自創＝某類工具／行為的熟練度達標後解鎖的進階效果，皆在預設效果池內。 | 與「裝備=工具、工具熟練度」一致；利：閉環；弊：名稱可自訂、效果不可憑空。 |

**組員推薦**：**A＋E**（組合既有模組 ＋ 技能即工具熟練度延伸）。**原因**：A 保證「不可無中生有」、技能皆為效果模組組合；E 與設計關鍵字「裝備=工具、工具熟練度」形成閉環，自創感來自「選組合＋熟練度解鎖」而非憑空造規則；敘事上可包裝為「門派／師傅傳授的仍是模組內的組合」，符合嚴謹架構。

主管決斷：ABCE。

### 7. 敘事呈現形式（「數據→文字」）

| 方案 | 簡述 | 參考／利弊 |
| --- | --- | --- |
| **A. 純滾動 log** | 所有敘事為一段段文字 append 到同一 log，可捲動查閱；無彈窗。 | 類 MUD／部分 Roguelike；利：極簡、重文字；弊：重要訊息可能被洗掉。 |
| **B. log＋重要事件彈窗** | 日常用 log，關鍵事件（戰鬥結果、任務完成、遭遇）用簡短彈窗或置頂一段。 | 利：輕畫面、重點突出；弊：需定義「重要」。 |
| **C. 分頻道 log（戰鬥／社交／系統）** | log 分欄或分頁：戰鬥、與 NPC 互動、系統訊息分開。 | 利：資訊不混雜；弊：介面略複雜。 |
| **D. 敘事以「地點／場景」為單位** | 進入一地或觸發一景，顯示該段文字（可長可短），再回到地圖或 log。 | 類視覺小說／段落式 MUD；利：沉浸感強；弊：節奏需設計。 |
| **E. 混合：常駐 log＋可展開的「當前情境」** | 常駐短 log；點擊當前位置或 NPC 可展開較長情境敘述。 | 利：輕操作、可深可淺；弊：需前後端配合展開時機。 |

**組員推薦**：**B＋E**（log＋重要事件彈窗 ＋ 常駐 log＋可展開當前情境）。**原因**：B 讓戰鬥結果、任務完成、遭遇等有明確呈現、不淹沒在滾動文字中，符合「重文字」又輕畫面；E 讓「點擊即展開」與「全物件可互動」一致，深度沉浸者可讀情境、輕度者只看常駐 log，兼顧手機輕操作與重體驗。

主管決斷：BCE。

### 8. 新手／引導

| 方案 | 簡述 | 參考／利弊 |
| --- | --- | --- |
| **A. 無教學，出生即進世界** | 不主動教學，僅可選「幫助」或「操作說明」頁；靠介面自解釋與探索。 | 類部分 MUD／硬核；利：沉浸、不打断；弊：門檻高。 |
| **B. 極短強制教學（僅操作）** | 僅教移動、點擊、走／跑，約 1～2 分鐘，其餘自探索。 | 利：低門檻、不冗長；弊：經濟／戰鬥等需另尋說明。 |
| **C. 首登分段提示** | 第一次登入時，在關鍵時機（第一次移動、第一次遇 NPC）彈一次短提示，可關閉。 | 利：情境化、不打斷整體；弊：需設計觸發點。 |
| **D. 可重複查閱的「世界規則」與「操作」** | 無強制流程，但主選單或固定按鈕可開「規則簡述」「操作說明」供隨時查。 | 利：尊重「重體驗」、需要再查；弊：不查的人可能一直不懂。 |
| **E. 出生地 NPC 口頭指引** | 出生點有 1～2 個 NPC 可對話，對話內容即簡短指引與世界觀；無強制步驟。 | 利：符合「重文字」、有世界感；弊：玩家可跳過不讀。 |

**組員推薦**：**B＋D＋E**（極短操作教學 ＋ 可重複查閱規則／操作 ＋ 出生地 NPC 口頭指引）。**原因**：B 只教移動／點擊／走跑、約 1～2 分鐘，門檻低且不違背「重體驗輕操作」；D 尊重「需要再查」、不強制灌輸；E 用文字與 NPC 傳達世界觀與指引，符合「重文字」且與萬名 NPC 世界感一致；三者並存不冗長、又保留沉浸與自探索空間。

主管決斷：A。
