## 二、第一版不做（留後續）

| 區塊 | 內容 | 說明 |
| --- | --- | --- |
| **詞盤進階** | 造句→祭煉成敘事、系統判定語意、自創技能授名、陣列 | 詞盤彙整 §9.4；可先做「固定技能表」或少量敘事範本 |
| **詞盤完整** | 剝名（非百分百成功、屬性波動）、竅穴打通／入詞／連線、裝備附魔槽與強化階 | 詞盤彙整 §9；第一版可僅「鎂＋簡單裝備欄位」 |
| **人物屬性完整** | 出身×念紋加權、竅穴／連線對照表完整計算、戰鬥結算公式完整 | 人物屬性彙整；第一版可用簡化屬性或固定值 |
| **觀測坍縮完整** | 多重疊加態故事線、過去態／現在態／未來態分層坍縮 | 設計關鍵字；第一版可「觀測→回推單一狀態」最小版 |
| **社會／門派** | 門派、氏族、城鎮層級、登錄到工會／世界詞盤 | 關鍵字有、未細化；第一版可不做 |
| **時間系統** | 日夜／季節、時間綁定產出、事件與時間對接 | KEYWORDS_ASSESSMENT；第一版可無或僅世界 tick |
| **死亡／懲罰細則** | 死亡後短暫離線化、完整懲罰表 | 5a；第一版可僅「重生＋扣鎂」 |
| **政治／生態引擎** | 若 1.1.6 只做經濟，政治／生態可後加 | 決策 004 |
| **下線變 NPC** | 玩家下線後角色由後端驅動為 NPC | 設計關鍵字；第一版可僅「下線即離線」 |

## 三、優先順序建議（組員）

建議實作順序（可依主管調整）：

1. **後端 scaffold + WebSocket + SQLite schema**（1.1、1.8.3）
2. **地圖格點與視野查詢**（1.2）、**移動與事件**（1.3）
3. **前端 Canvas 地圖 + 符號 + 移動操作**（1.9.1～1.9.3）
4. **實體與插頭插座最小集**（1.5、1.6）、**觀測與坍縮最小版**（1.4）
5. **戰鬥最小版**（1.7）、**經濟最小閉環**（1.8.1～1.8.2）
6. **詞盤／對話／log 前端區塊**（1.9.4）、**經濟引擎 goroutine**（1.1.6）

主管勾選或改寫後，組員可依此清單拆成具體任務（issue／ticket）。
