# 人物屬性建議（組員端大數據）

主管未定「人物屬性」細項時，組員依設計關鍵字與詞盤系統提出**多項方案**供主管決斷。屬性需與**竅穴／連線、裝備＝工具、戰鬥全自動、後端唯一真理、玩家＝NPC** 對齊。

---

## 一、屬性架構方案（3～5 種）

| 方案 | 簡述 | 與本專案對齊／利弊 |
|------|------|---------------------|
| **A. 詞盤驅動型** | 人物無獨立「力敏體」等裸值，**全部由內盤與裝備推導**：竅穴打通→普通加成（體質、氣脈等）、詞元入竅→特殊加成（火系、疾速等）、連線→額外加成；裝備附魔／強化再疊加。戰鬥結算時後端彙總「內盤＋裝備」得出有效屬性。 | 與詞盤系統完全一致；利：單一來源、永序守衡、無裸值膨脹；弊：需先定竅穴／連線規則表。 |
| **B. 雙軌型（基礎＋詞盤）** | 角色有**少量基礎屬性**（例：體質、氣脈、靈敏，3～5 項），來自種族／出身或初始分配；其餘由竅穴、連線、裝備疊加。戰鬥＝基礎＋詞盤加成＋裝備。 | 利：有「天生」差異、敘事好寫；弊：基礎與詞盤邊界需定義清楚，避免雙通膨。 |
| **C. 經典六維縮簡** | 力／敏／體／智／意／運 等 3～6 維，數值由升級、裝備、詞盤加成共同影響。 | 類傳統 MMO／MUD；利：直觀、易對照他作；弊：與「裝備=工具、無職業」需對齊，避免「智」綁法師等職業化。 |
| **D. 戰鬥向＋生活向分離** | **戰鬥用**：傷害／承傷／命中／閃避／速度 等結算用維度；**生活用**：採集效率、負重、交涉權重 等。兩類可來自同一底層（詞盤＋裝備）不同換算。 | 利：戰鬥 log 與田忌賽馬直接對應；生活向可支援經營／建造；弊：維度變多，需後端表驅動。 |
| **E. 純效果池（無具名屬性）** | 不設「體質」「火抗」等具名屬性，僅設**效果標籤與數值**（如 `resist:fire +10%`、`speed +5%`）；全部由竅穴、連線、裝備、技能貢獻。 | 利：極簡、全由詞盤與裝備驅動；弊：前端／敘事若需「一句話描述角色強在哪」較需轉譯。 |

**組員推薦**：**A** 或 **B**。  
**原因**：A 與詞盤「打通→入詞→連線、後端計算認證」完全閉環，無多餘裸值，永序守衡最佳。若希望角色有「出身／種族」帶來的天生差異（含萬名 NPC 多差異個性），選 B，基礎項建議極少（3 項以內）、且不隨升級膨脹，成長主線仍放在詞盤與裝備。

**主管決斷**：**A＋B**。理由：本案已定有**精神念紋**（詞盤系統），念紋決定與哪類詞元共振、天賦方向；雙軌的「少量基礎屬性」可與念紋對接——例如由念紋類型給予少量基礎值或天賦係數，其餘由竅穴打通／入詞／連線與裝備疊加，兼顧天生差異與詞盤驅動。

## 二、建議屬性項（若採 A 或 B 之具體名稱）

以下名稱與詞盤腦洞補完用語對齊，供後端表與敘事使用。

### 2.1 可由「竅穴打通」產生的普通加成（基礎型）

| 名稱 | 說明 | 戰鬥／生活 |
|------|------|------------|
| **體質** | 承傷、負重、耐性 | 戰鬥＋生活 |
| **氣脈** | 續戰、技能／行動消耗、回復 | 戰鬥 |
| **靈敏** | 命中、閃避、行動序 | 戰鬥 |

（可再收斂為 2 項，例如「體質」「氣脈」，靈敏併入裝備或詞元「疾速」類。）

### 2.2 可由「詞元入竅」產生的特殊加成（屬性型）

與剝名／詞元屬性對齊，例如：

- **火**：火系傷害、火抗、火系技能增幅  
- **冰**：冰系傷害、冰抗、遲緩相關  
- **堅韌**：物防、破防抗性  
- **疾速**：行動序、跑速、攻速  
- **靈敏**（若未放在普通）：命中、閃避  

（實際項目依「允許的詞元屬性表」由後端定義；此處僅示範。）

### 2.3 戰鬥結算用維度（可全部由 2.1＋2.2＋裝備 推導）

- **傷害輸出**（依技能與詞元屬性）  
- **承傷／減傷**（體質、堅韌、各抗性）  
- **命中／閃避**（靈敏、疾速、裝備）  
- **行動序／速度**（氣脈、疾速、裝備）  

不建議再多一層「攻擊力／防禦力」裸值，除非明確定義為上述維度的加總或加權，避免與詞盤、裝備重複計價。

## 三、與他系統的對齊

| 系統 | 對齊要點 |
|------|----------|
| **詞盤** | 普通加成＝竅穴打通；特殊加成＝詞元入竅；連線＝額外或套裝效果。全部由後端計算認證。 |
| **裝備＝工具** | 裝備提供「工具維度」加成（如鋒利→傷害、堅韌→承傷）；與人物屬性相加或加權後進入戰鬥結算。工具熟練度可視為「裝備維度」的係數或解鎖門檻。 |
| **戰鬥全自動** | 分秒輸贏＝f(有效屬性, 技能, 裝備, 隨機因子)；有效屬性＝內盤加成＋裝備，無前端即時操作。 |
| **玩家＝NPC** | 同一套屬性架構與計算公式；NPC 差異可來自「基礎值（若 B）＋內盤配置＋裝備」不同，支援萬名多差異個性。 |
| **永序守衡** | 屬性有來源（竅穴、詞元、裝備），無「憑空加點」；成長＝擴充內盤與裝備，後端唯一真理。 |

---

## 四、建議下一步

- 主管擇定 **架構方案（A～E 或混合）** 後，可將「普通加成項」「特殊加成項（詞元屬性表）」「戰鬥結算維度」收進設計常數或規格表，供後端與詞盤細規對接。
- 若採 A 或 B，建議在詞盤細規中補齊：**竅穴與普通加成對照表**、**詞元屬性與特殊加成對照表**、**連線加成規則**，以利後端計算認證。
