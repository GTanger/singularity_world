# 經濟彙整（奇點世界）

> 本專案經濟相關設定與參考**單一彙整**於此。涵蓋：概念釐清、水龍頭與排水口、他作遊戲對照、**鎂的產消閉環**、單一真人×萬名 NPC 市場設計、專案經濟規劃建議與實作階段。與 [詞盤彙整](詞盤彙整.md)（§九竅穴／詞元／語意）、[決策 002](decisions/002_sockets_plugs_semantic.md)、[KEYWORDS_ASSESSMENT](../KEYWORDS_ASSESSMENT.md) 對齊。

**情境**：可能僅 1 名真人玩家，萬名以上擬真人 NPC；鎂為單一通用貨幣；設計取向為**自由＝鎂在市場流動**，**不對玩家步步扣款**。

---

## 一、概念釐清

### 1.1 市場經濟（Market Economy）

資源與勞務主要由**私人或個體**擁有與支配；**價格與產出**由市場**供給與需求**決定，而非單一中央計畫。要點：自願交換、競爭、供需決定價格與數量。**在遊戲中**：物品與貨幣的擁有、定價、交易由**多個參與者**的決策與交易形成，而非僅由開發者價表決定。

### 1.2 自由市場經濟（Free Market Economy）

理論上**系統干預極少**，價格與產出幾乎完全由**供需**決定。要點：自由訂價、自由交易；價格由**買賣雙方自願合意**形成。**在遊戲中**：開發者只提供規則與基礎設施，**不強制每一步扣款或固定物價**；價格由 NPC 與玩家的掛單、議價、成交自然形成。

### 1.3 自由交易（Free Trade）

**自願**在雙方合意下交換物品、服務或貨幣；無強制、無單一壟斷定價。**在遊戲中**：玩家與 NPC 可提出買／賣意圖、出價、還價、成交；流動來自**意願與決策**，非「系統強制收費」。

### 1.4 真實經濟感（遊戲語境）

- **稀缺**：物品與貨幣有有限來源與消耗（修裝、附魔、死亡、交易），故有供需。  
- **自願交換**：價格與數量由參與者行為驅動，而非僅腳本價表。  
- **流動可見**：有**鎂在市場裡流動**（誰買誰賣、誰賺誰花）。  
- **來源與去向可追溯**：符合「永序守衡」——不無中生有，產出與回收由後端建模、可記帳。

**本專案**：「真實經濟」＝稀缺、自願交換、流動、產消閉環；「自由」＝流動來自 NPC／玩家行為，而非處處扣款與固定價表。

## 二、水龍頭與排水口、玩家驅動

### 2.1 Faucet（水龍頭）與 Sink（排水口）

- **Faucet**：遊戲內**創造新貨幣**的來源（如任務獎勵、薪餉、事件、配給）；玩家間交易只移動既有貨幣，不創造新幣。  
- **Sink**：**回收貨幣**的機制（手續費、稅、修裝、傳送、維護等）。若只有 faucet 而無足夠 sink，會通膨。  
- **設計鐵律**：**「Stuff in ≈ Stuff out」**——注入量與回收量大致平衡，經濟才不會崩盤或緊縮過度。本專案中機構只做注入與基礎回收，**不對玩家個人強制扣款**；平衡在背景以據點稅與自願消費達成。

### 2.2 玩家驅動（Player-Driven）與本專案

他作：定價與交易由玩家掛單形成；生產與消耗由玩家製造、採集、戰鬥、死亡驅動。**本專案**：定價與交易由**萬名 NPC 與玩家**的交易意圖與成交形成；NPC 也參與生產、消費、詞盤（竅穴、祭煉、附魔），市場深度由 NPC 行為支撐，單人也能感受「市場在活著」。

### 2.3 實體貨幣與遊戲幣邊界（參考）

- **EVE 模式**：僅允許「實體貨幣 → PLEX → 玩家市場換 ISK」；禁止 RMT。  
- **共通**：自由經濟不等於完全不管；貨幣創造與回收仍由設計控制，自由體現在**玩家定價、分工、風險與報酬**（本專案為 NPC＋玩家共同驅動）。

## 三、各遊戲如何維持經濟活動

### 3.1 維持經濟活動的共通要素

| 要素 | 說明 |
|------|------|
| **有供給** | 有人（或 NPC）產出物品、服務、貨幣；來源有限（時間綁定、配方、掉落），避免無限刷。 |
| **有需求** | 有人需要購買、消耗、升級、替換；需求來自戰鬥損耗、成長、任務等。 |
| **有價格發現** | 價格由掛單、撮合、供需或 NPC 行為驅動，而非僅固定價表。 |
| **有流動** | 貨幣與物品在參與者之間**實際轉手**。 |
| **有破壞／回收** | 物品損壞、消耗、回收，貨幣被 sink，形成持續需求與稀缺。 |

### 3.2 各遊戲對照摘要

| 遊戲 | 誰生產 | 誰消費 | 誰定價 | 經濟如何持續動 |
|------|--------|--------|--------|----------------|
| **EVE** | 玩家採礦、製造、任務領 ISK | 玩家買船買裝、戰爭摧毀、繳稅與手續費 | 玩家掛買賣單撮合；僅 PLEX 官方介入 | 單一宇宙：生產在安全區、摧毀在低／零安；生產—破壞循環；專職經濟學家與月報。 |
| **Albion** | 玩家採集、製造；打怪掉銀、黑市收購裝備 | 玩家修裝、買裝、繳稅、維護；全掉落消耗裝備 | 玩家掛單＋黑市動態收購價 | 全掉落＋黑市：裝備來自玩家→黑市→當怪掉落；銀由怪掉與黑市抽成調節。 |
| **PoE** | 打怪掉通貨與裝備；無金幣 | 玩家 craft、買裝；高階 craft 大量消耗通貨 | 玩家間以物易物（通貨為媒介） | 通貨即物品；通貨向左流、裝備向右流；craft 為主要 sink。 |
| **UO** | 玩家採集、製造；部分 NPC 模擬供需 | 玩家與 NPC 購買、消耗 | 早期部分 NPC 動態定價；後補貼與固定 | 教訓：產消邊界清晰、適度回收；NPC 行為需有上限與記帳。 |
| **OSRS** | 打怪、任務、賣出給系統 | 玩家買裝、修裝、消費；GE 稅回收 | GE 掛單撮合＋1% 稅 | 單一交易所＋稅。 |
| **New World** | 採集、製造、任務、事件 | 房產稅、交易站費、精煉費、購買 | 玩家掛單＋據點稅率 | 領地與房產稅、服務費綁據點。 |
| **單人／NPC 驅動** | NPC 或系統依規則產出；玩家可參與 | NPC 與玩家消費；NPC 有需求表 | 供需模擬或 NPC 掛單與議價 | 即使單人，市場因 NPC 間交易與玩家參與而持續動。 |

### 3.3 腳本經濟 vs 湧現經濟

- **腳本經濟**：物品來源、價格、回收由開發者寫死或固定表；稀缺人工；玩家主要與系統互動。→ 易控制，經濟感較弱。  
- **湧現經濟**：真實稀缺；價格由參與者供需與交易形成；有套利、價差、行為反饋。→ 真實經濟感強。**本專案目標趨近湧現**，參與者包含萬名 NPC，故以「NPC 行為驅動價格與流動」。

## 四、鎂的產消閉環

鎂＝世界通用貨幣、經機構賦權後流通。**不可無中生有**；產出與回收由後端建模，形成閉環。設計取向：真實經濟感來自**鎂在市場流動**（萬名 NPC 與玩家主動買賣）；機構僅負責注入與基礎回收，**不對玩家個人行為強制扣款**。

### 4.1 鎂的產出（來源）

| 來源類型 | 說明 | 備註 |
| --- | --- | --- |
| **任務／委託報酬** | NPC 或機構發放任務，完成後由機構結算發放鎂 | 報酬數由任務難度／類型查表 |
| **交易所得** | 玩家／NPC 販售物品、服務，買方支付鎂 | 物價可依供需或 NPC 行為 |
| **據點／門派薪餉** | 加入門派、據點後定期領取（若設計有） | 可綁時間或貢獻度 |
| **事件獎勵** | 世界事件、劇情節點完成後獎勵 | 一次性或稀有 |
| **初始／重生配給** | 角色創建或重生時少量配給（可選） | 避免開局無鎂，需設上限 |

※ 鎂**不從剝名取得**；剝名所得為帶屬性的詞元，用於附魔／強化／入竅。  
※ 剝名：非百分百成功；所得詞元屬性可有波動，由後端表或隨機區間定義。

### 4.2 鎂的回收（消耗）

| 消耗類型 | 說明 | 備註 |
| --- | --- | --- |
| **交易支出** | 向 NPC／玩家購買物品、服務 | 與產出「交易所得」對應 |
| **附魔／強化** | 裝備附魔、強化階提升時消耗鎂（經機構或儀式賦權） | 詞盤系統核心消耗 |
| **自創技能登錄** | 登錄到工會／門派／世界詞盤時交付鎂 | 可選 |
| **稅／維護費** | 據點維護、城鎮稅、傳送費等；**以據點／NPC 為主**，玩家可完全不碰 | 見 KEYWORDS_ASSESSMENT |
| **死亡／懲罰** | 死亡扣部分鎂（依主管決斷） | 見 KEYWORDS_ASSESSMENT 死亡方案 |

※ **技能不消耗鎂**；技能執行僅消耗氣脈／行動等。  
※ 裝備為附魔與強化的**重點消耗對象**。

### 4.3 後端建模要點

- 每筆鎂的增減皆可**記帳**（誰、何時、來源／消耗類型、數量），利於除錯與經濟平衡。  
- 總量可設上限或不上限；若設上限，回收不足時可透過時間綁定產出或任務報酬調節。  
- NPC 持有鎂：萬名 NPC 可依身份／門派有初始或週期產出與消耗，與玩家共用同一套產消類型。

## 五、本專案：單一真人 × 萬名擬真人 NPC

### 5.1 核心原則

- **自由＝流動**：經濟真實感來自**鎂在 NPC 與玩家之間、NPC 與 NPC 之間**真實買賣、交換、決策，而非靠收費扣款制度強制平衡。  
- **萬名 NPC 的定位**：像玩家一樣在買賣、囤貨、投機、祭煉；其交易意圖與成交構成市場深度。  
- **機構與回收**：機構只負責**新鎂注入**（任務、薪餉、事件、配給）與**基礎回收**（僅對 NPC 據點收稅，玩家可完全不碰）；**絕不對玩家個人行為強制扣款**。玩家鎂的增減都來自與 NPC 或玩家的**真實交易**。

### 5.2 萬名 NPC 的玩家行為（讓鎂流動）

- **主動生成交易意圖**：每遊戲日（或每 N tick），NPC 依念紋、出身、當前庫存生成 1～3 筆交易意圖（買／賣／價）；對同房或鄰房 NPC 或玩家發起；WebSocket 推播可成交提案，玩家一鍵同意即成交。  
- **NPC 與 NPC 交易**：視野外以事件日誌紀錄隱形交易（扣鎂、轉物、寫 log）；視野內跑完整流程，可選推播。市場即使玩家沒上線也動。  
- **NPC 參與詞盤**：收集詞元、打通竅穴、入詞、連線、祭煉、自創技能、附魔，用**自己持有的鎂**買材料、買詞元、支付登錄費；形成 NPC 間詞元與鎂雙向流動。  
- **玩家與同房 NPC 交易**：玩家點擊同房 NPC，後端回傳「此 NPC 目前想買／想賣」即時清單（[插頭插座](decisions/002_sockets_plugs_semantic.md)：Trade 為插頭，NPC 買賣意圖為插座）；一鍵同意即成交。  
- **NPC 依市場狀況改變行為**：某詞元或物品熱門→提高收購價或自己去蒐集轉賣；供過於求→壓價或轉賣他物。價格自然波動，可設合理上下限防崩盤。  
- **下線玩家角色變 NPC**：執行玩家最後設定的簡單交易策略（如「一直收火詞元」「賣多餘裝備」），與其他 NPC 成交；上線可看「角色幫你賺了多少、賣了什麼」。  
- **交易所／大廳**：顯示「最近 24 小時交易熱門 Top 10」與「市場趨勢」（詞元價格漲跌），單人也能感受市場脈動。

### 5.3 機構只做兩件事

- **新鎂注入**：任務／委託報酬、據點薪餉、事件獎勵、初始或重生配給；NPC 也從機構領鎂（如店員、工匠薪餉）。  
- **基礎回收**：稅只對 NPC 據點收；不對玩家個人買賣、移動、戰鬥強制扣款。玩家可選服務（傳送、附魔、登錄技能）為**主動花鎂**，非系統自動扣。

### 5.4 後端建模建議（摘要）

1. NPC 交易意圖：每遊戲日或每 N tick，依念紋＋出身＋庫存為 NPC（或抽樣視野外）生成 1～3 筆買／賣意圖，寫入待成交池或對同房／鄰房提案。  
2. NPC-NPC 成交：視野外＝事件日誌；視野內＝完整流程＋可選推播；成交更新雙方鎂與庫存、寫入事件日誌。  
3. 玩家點擊 NPC 的 Trade：回傳該 NPC 當前「想買／想賣」清單，玩家選擇後一鍵成交。  
4. 市場狀況反饋：後端維護近期成交價／成交量或簡化指數，供 NPC 依市場調價／調行為；交易所大廳顯示熱門與趨勢。  
5. 下線角色交易策略：玩家可設定簡單策略，下線後角色變 NPC 依此生成意圖並參與成交；上線可查下線期間收支與成交紀錄。  
6. 機構帳本與平衡：機構發放與據點稅等回收記帳，用於長期平衡調參；不增加對玩家個人的強制扣款。

## 六、專案經濟規劃建議

### 6.1 維持經濟活動的五大支柱

| 支柱 | 建議 |
|------|------|
| **1. 供給** | 機構為唯一新鎂注入源；物品與詞元由採集、剝名、NPC 製造、任務產出，可時間綁定或冷卻防無限刷。 |
| **2. 需求** | 萬名 NPC 具需求表（修裝、祭煉、附魔、收購詞元／材料）；NPC 之間也互相買賣，玩家沒上線市場仍動。 |
| **3. 價格發現** | 由 NPC 行為驅動：每遊戲日／tick 生成 1～3 筆交易意圖，依念紋、出身、庫存與市場狀況調整；無固定全局價表，可設上下限。 |
| **4. 流動** | 鎂與物品在 NPC↔NPC、NPC↔玩家之間實際轉手；視野外事件日誌；下線角色可執行交易策略；交易所顯示熱門與趨勢。 |
| **5. 回收** | 機構僅做基礎回收（據點稅以 NPC 為主）；不對玩家個人強制扣款；回收來自自願消費與據點營運；長期注入≈回收由後端記帳與調參達成。 |

### 6.2 實作優先順序

1. **第一階段（最小可玩經濟）**：機構發放鎂（任務、初始配給）；NPC 具「想買／想賣」清單（常數表或簡單公式），玩家點同房 NPC 可交易（插頭插座 Trade）；回收僅交易支出與自願附魔／登錄；暫不實作據點稅。  
2. **第二階段（流動與深度）**：NPC 主動生成交易意圖；NPC 與 NPC 視野外事件日誌成交；NPC 參與詞盤（收購詞元、消費附魔／登錄）；價格依庫存或簡單市場指數微調；交易所／大廳顯示熱門與趨勢。  
3. **第三階段（穩定與擴充）**：據點稅（僅 NPC 據點）、時間綁定產出與冷卻；後端記帳與「機構發放 vs 全服回收」分析，調參防通膨／緊縮；下線角色交易策略與上線收支查詢。

### 6.3 風險與邊界

- **NPC 計算量**：萬名 NPC 可改為抽樣或僅視野內＋鄰近即時生成意圖，其餘簡化事件日誌或週期批次。  
- **價格極端**：保留上下限或阻尼係數，避免單一資源暴漲暴跌。  
- **單人無對手方**：至少一類玩家可產出品項（剝名、採集、任務道具）有 NPC 常態收購；至少一類玩家需要的品項有 NPC 常態供貨，避免有市無量或有價無市。

## 七、參考：他專案經濟收斂要點

- **通膨控制**：產出有來源、回收成閉環（稅／維護費＋裝備損壞＋傳送付費＋據點維護）、可驗證。  
- **玩家間交易**：P2P 靠近給予金錢或物品；市集／拍賣可系統或據點抽成進入閉環（參考 Albion／EVE／Wakfu）。  
- **物價**：供需＋世界事件影響；首版可部分固定價。  
- **產出—消耗**：禁止無中生有；裝備＝工具，工具熟練度，全身多部位。
