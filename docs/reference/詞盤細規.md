# 詞盤細規（後端可實作）

> 依據 [詞盤系統_主管設想](../詞盤系統_主管設想.md) 補齊，供後端唯一真理與數值計算使用。與 [人物屬性細規](人物屬性細規.md) 共用對照表。

---

## 一、鎂的產消閉環

鎂＝世界通用貨幣、僅經瑞式族人賦權後流通。**不可無中生有**；產出與回收由後端建模，形成閉環。

### 1.1 鎂的產出（來源）

| 來源類型 | 說明 | 備註 |
| --- | --- | --- |
| **任務／委託報酬** | NPC 或機構發放任務，完成後由瑞式或機構結算發放鎂 | 報酬數由任務難度／類型查表 |
| **交易所得** | 玩家／NPC 販售物品、服務，買方支付鎂 | 物價可依供需或固定表 |
| **據點／門派薪餉** | 加入門派、據點後定期領取（若設計有） | 可綁時間或貢獻度 |
| **事件獎勵** | 世界事件、劇情節點完成後獎勵 | 一次性或稀有 |
| **初始／重生配給** | 角色創建或重生時少量配給（可選） | 避免開局無鎂，需設上限 |

※ 鎂**不從剝名取得**；剝名所得為帶屬性的詞元，用於附魔／強化／入竅。  
※ **剝名**：非百分百成功，後端可依目標類型／難度設成功率；剝名成功時，所得詞元的**屬性也有波動**（同一來源可能得到不同強度或副屬性），由後端表或隨機區間定義。

### 1.2 鎂的回收（消耗）

| 消耗類型 | 說明 | 備註 |
| --- | --- | --- |
| **交易支出** | 向 NPC／玩家購買物品、服務 | 與產出「交易所得」對應 |
| **附魔／強化** | **裝備**附魔、**裝備**強化階提升時消耗鎂（經瑞式或儀式賦權） | 詞盤系統核心消耗；裝備為強化與附魔的重點 |
| **自創技能登錄** | 將自創技能登錄到工會／門派／世界詞盤時交付鎂 | 可選 |
| **稅／維護費** | 據點維護、城鎮稅、傳送費等（若設計有） | KEYWORDS_ASSESSMENT 推薦 |
| **死亡／懲罰** | 死亡扣部分鎂（依主管決斷） | 見 KEYWORDS_ASSESSMENT 死亡方案 |

※ **技能不消耗鎂**；技能執行僅消耗氣脈／行動等，非投擲金錢鏢。  
※ **裝備**為強化與附魔的**重點消耗對象**：鎂的回收以「裝備附魔、裝備強化階提升」為核心場景。

### 1.3 後端建模要點

- 每筆鎂的增減皆可**記帳**（誰、何時、來源／消耗類型、數量），利於除錯與經濟平衡。
- 總量可設**上限**或**不設上限**；若設上限，回收不足時可透過「時間綁定產出」或任務報酬調節。
- NPC 持有鎂：萬名 NPC 可依身份／門派有初始或週期產出與消耗，與玩家共用同一套產消類型。

## 二、竅穴與普通加成、詞元屬性與特殊加成對照表

與 [人物屬性細規](人物屬性細規.md) 共用。後端依此表計算「竅穴打通→普通加成」「詞元入竅→特殊加成」「連線→額外加成」。

### 2.1 竅穴打通 → 普通加成（基礎型）

| 竅穴類型（可擴充） | 打通後普通加成項 | 數值／效果說明（例） |
| --- | --- | --- |
| 體竅 | 體質 | 承傷減免、負重、耐性；數值＋N 或百分比 |
| 氣竅 | 氣脈 | 續戰、技能／行動消耗、回復；數值＋N 或百分比 |
| 敏竅 | 靈敏 | 命中、閃避、行動序；數值＋N 或百分比 |

- **竅穴總數／上限**：由後端常數或角色模板定義（例：每人 8～12 竅，可依成長開放）。
- **打通條件**：可設為消耗鎂、任務、或達到某條件後打通；具體由專案再定。

### 2.2 詞元屬性 → 特殊加成（詞元入竅後）

與剝名所得、蒐集詞元的「屬性」對齊。入竅後該竅提供對應特殊加成。

| 詞元屬性（可擴充） | 特殊加成效果（例） |
| --- | --- |
| 火 | 火系傷害＋、火抗＋、火系技能增幅 |
| 冰 | 冰系傷害＋、冰抗＋、遲緩／凍結相關 |
| 堅韌 | 物防＋、破防抗性 |
| 疾速 | 行動序、跑速、攻速＋ |
| 靈敏 | 命中、閃避＋（若未併入普通） |
| 記憶 | 技能冷卻、敘事相關（可選） |

- **詞元屬性表**：後端維護一份「允許的詞元屬性」清單；剝名／蒐集得到的詞元帶有其中一類或多類標籤。
- **同竅多詞元**：若一竅可容納多個詞元，規則需另定（例：僅取最高、或疊加上限）。

### 2.3 連線加成規則（示範）

- **連線**：多個詞元竅穴依規則（經脈拓樸、星圖等）形成連線；連線成立時觸發**額外加成**。
- **規則例**：同一「屬性系」的竅穴達 N 個以上連通 → 觸發該系套裝加成（如火系 3 竅連線 → 火傷＋5％）。
- **後端**：可定義「連線模板」表（哪些竅穴編號相連、觸發何種加成 ID），戰鬥／結算時查表彙總。

## 三、系統判定語意邊界（造句／祭煉）

造句→祭煉成敘事→**系統判定**中文邏輯、語順、語意→通過則生成屬性加成或技能效果、並授予技能名（2～4 字）。

### 3.1 判定範圍

| 項目 | 說明 |
| --- | --- |
| **輸入** | 玩家／NPC 提交的「造句」＝一串詞元或一句敘事（如「我以火擊之」）。 |
| **輸出** | 通過：技能名（系統萃取 2～4 字）、效果類型（單體火傷、連擊、強化等）、數值或係數。不通過：不生成技能、可回傳原因（選用）。 |
| **邊界** | 僅接受**允許的詞元集合**內之組合；語順、語意由規則表或關鍵字匹配決定，不開放任意自然語言。 |

### 3.2 允許的詞元集合（示範）

- 後端維護「可參與造句的詞元 ID 表」及「詞元→語意標籤」對照（如：火、擊、單體、範圍、己、跑）。
- **敘事範本**（例）：`[己][動詞][目標][屬性]` → 「我以火擊之」對應 己＋以＋火＋擊之；匹配成功則輸出技能「火球術」、效果單體火傷。
- **擴充**：新技能可透過「登錄」加入範本表，供他人學習或系統判定時查詢。

### 3.3 邏輯／語順規則（示範）

- **語順**：主詞＋動詞＋目標＋修飾；不符合則不通過。
- **語意**：關鍵字對應效果類型（火→火傷、跑→速度、連擊→多段）；未在表內則不通過或歸為「通用」。
- **平衡**：同一造句僅對應一組技能名與效果；重複提交同一造句不重複授名（已登錄則回傳已有技能名）。

### 3.4 系統授名（2～4 字）

- 技能名由後端**萃取簡名**，不開放玩家自訂。
- 對照表示範：`敘事 ID / 造句關鍵 → 技能名`（如「我以火擊之」→「火球術」，「我以烈火連擊之」→「烈火連擊」，「我會跑」→「疾走」）。
- 描述＝完整敘事＋發明者＋效果（可存於技能資料，供前端文字敘述）。

## 四、與人物屬性細規的對接

- **普通加成項**：§2.1 與人物屬性細規的「基礎項」分離：§2.1 為竅穴打通帶來的**成長型**普通加成；人物屬性的「基礎」為念紋／出身帶來的**天生**少量值，兩者相加後再與 §2.2 特殊加成、§2.3 連線、裝備疊加。
- **戰鬥結算**：有效屬性 ＝ 基礎（出身×念紋 加權）＋ 竅穴普通加成 ＋ 詞元特殊加成 ＋ 連線加成 ＋ 裝備；後端依此順序彙總後代入戰鬥公式。
