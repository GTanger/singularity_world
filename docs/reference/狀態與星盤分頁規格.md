# 狀態與星盤分頁規格（人物彈窗 UI）

> 人物點擊彈窗之【狀態】與【星盤】分頁的 UI 結構與用語。收斂自 [gemini chat_狀態與星盤分頁](gemini%20chat_狀態與星盤分頁.md)。對齊 [人物角色模板](人物角色模板.md) §八、[人物屬性彙整](人物屬性彙整.md)、[361拓撲系統規格](361拓撲系統規格.md)。

**原則**：UI 只顯示後端已真實算出的數據；不顯示尚未實作的戰鬥屬性（閃避率、暴擊率等假資料）。

## 一、分頁導航

人物彈窗四個分頁標籤為：

| 分頁 | 說明 |
|------|------|
| **狀態** | 命途、本源、維度、四相、持有（見 §二） |
| **裝備** | 17 槽位列表，見 [裝備分頁規格](裝備分頁規格.md) |
| **技能** | 運轉功法、實戰招式池、語境推演；見 [技能分頁規格](技能分頁規格.md) |
| **星盤** | 361 竅穴網路之層級式呈現（原「經脈」分頁，見 §三） |

## 二、【狀態】分頁

採用純文字排版，分區如下。資料來源：後端實體之 SoulSeed 展開、體敏氣、四項資源公式、鎂。

| 區塊 | 內容 | 資料綁定與備註 |
|------|------|----------------|
| **命途** | 稱謂／身份 | 後端 **display_title**；空則前端顯示「無名之輩」。未來門派／稱號直接更新此欄位。 |
| **本源** | 一句話描述 | 後端 **GenerateOriginSentence(amp, freq, phase)** 產出 **origin_sentence** 傳前端；**絕不**傳三軸數字。區間→形容詞（霸道、幽微、渾厚、洞察、順流、混沌）拼句。見 §五.1。 |
| **維度** | 體質｜氣脈｜靈敏 | 有效體質、有效氣脈、有效靈敏（外顯三維） |
| **四相** | 氣血、內力、精神、體力 | 四項資源以「四相」統稱；顯示 當前/最大；條寬＝當前/最大×100％。 |
| **持有** | 鎂 | 鎂：{Magnesium} |

**用語約定**：

- **維度**：對應「體質、氣脈、靈敏」三維，與 361 之宇宙學命名一致。
- **四相**：氣血、內力、精神、體力四項資源之統稱（東方古典語感，不對「資源」等詞）。
- **本源**：三軸信號光譜的敘事化呈現，僅一句話語感，不暴露浮點數。

## 三、【星盤】分頁

361 節點網路之 UI 呈現；命名取「星盤」：生之奇點為中心，二十主樞為星系，五藍為衛星，十二綠為星環。**UI 上之【星盤】分頁即內盤（人身竅穴網路）之層級式呈現**——與詞盤彙整之「內盤」概念同一，僅以星盤命名與結構對應 361 拓撲。為避免資訊過載，採**漸進式揭露**與**層級式選單**。

### 3.1 區塊一：宏觀進度（頂部固定）

- **[ 星盤貫通率 ]** 已貫通節點數 / 360（進度條）
- **[ 源始 ]** N000 生之奇點 **(已點亮)**  
  - 世界觀：角色存在即靈魂已降臨，奇點為能量起點，**唯一預設貫通**，不需消耗即點亮。  
  - 系統：N000 為拓撲圖之根節點，能量從此出發；其餘節點需透過內視／打通逐步點亮。

### 3.2 區塊二：二十主樞目錄（可摺疊）

列出 20 個橘色主樞，依 361 規格 §2.2 四大陣列分組（動能、塑能、維穩、特權）。每項可標示：

- 主樞名稱（如 [天極] N001）
- 貫通：x/17（該主樞下 5 藍＋12 綠）
- 適性：未知／極佳／良好（由該主樞對應之型 A Cost 推得；未探明為「未知」）
- 操作：[ 進入觀測 ]（展開區塊三）

### 3.3 區塊三：觀測儀（展開後之樹狀詳情）

點選某一主樞「進入觀測」後，顯示該主樞下 **5 藍（起承轉協合）＋12 綠（G01～G12）** 之樹狀結構。

- **關鍵**：綠點旁須綁定後端 **Topology Cost**（該連線阻力），顯示為 `(Cost: XX.XX)`。
- 未貫通之邏輯閘可標「迷霧籠罩」或「需先貫通前置」；按鈕（打通、煉化）可 MVP 先做空殼或 log。
- 結構對應 361 規格 §四、§五（五常邏輯閘、十二微型狀態與 3:2 陣列咬合）。

## 五、實作要點（已定調）

以下為主管決斷後之實作要點，實作時依此執行；欄位與 schema 見 [人物角色模板](人物角色模板.md) §3.3、§六與 db/schema。

### 5.1 本源（後端拼句）

- 後端實作 **GenerateOriginSentence(amp, freq, phase)**，輸入三軸浮點數，輸出一句話字串。
- 邏輯：將 [0.1~3.0]（能階）、[0.5~2.0]（時脈）、[-1.0~1.0]（相位）對應形容詞（如霸道、幽微、渾厚、洞察、順流、混沌），拼成一段話。
- 該句話以 **origin_sentence** 傳給前端；**絕不**將能階／時脈／相位之數字傳前端，以維持神祕感。

### 5.2 命途（預留稱謂）

- entities 表欄位 **display_title**（TEXT，選填）。空或 null 時前端顯示「無名之輩」；未來門派／稱號直接更新此欄位。

### 5.3 星盤貫通（骨架先行、儲存預留）

- entities 表欄位 **activated_nodes**（TEXT/JSON），預設 `["N000"]`。前端渲染星盤時，節點 ID 不在清單內則顯示「未貫通」或「迷霧」。第一版不實作打通流程；主樞適性先寫死或佔位。
- **名存實未知原則**：所有節點名稱恆顯，但 Cost、適性、屬性加成在節點**未被探明**前顯示 `[未知]`。完整定義見 [星盤迷霧與命盤機制規格](星盤迷霧與命盤機制規格.md)。

### 5.4 內盤／星盤命名

- 代碼邏輯層（變數、Service、註釋）：統一 **InnerPlate**（內盤）。使用者介面層（分頁標題、UI 文案）：統一 **星盤**。註釋標明：「星盤為內盤概念之 UI 呈現視圖」。

### 5.5 Cost 文字化（通暢度標籤）

星盤 UI 之邊權（型 A／B／C 之 Cost）以**語意標籤 + 精確值**呈現，取代純數據，以維持沉浸感。

**基準**：全域平均 Cost ≈ 10000 ÷ 760 ≈ 13.16。以此為中位劃分五級。

| 等級 | 閾值（Cost 範圍） | 標籤 | 色彩 | 語意 |
|------|-------------------|------|------|------|
| 1 | cost ≤ 7 | **暢流** | 綠 `#67c23a` | 能量幾無阻滯，極佳通道 |
| 2 | 7 < cost ≤ 11 | **順通** | 青 `#79bbff` | 阻力偏低，通行順暢 |
| 3 | 11 < cost ≤ 16 | **平穩** | 灰 `#8b949e` | 阻力適中，不特別優劣 |
| 4 | 16 < cost ≤ 21 | **滯澀** | 橙 `#e6a23c` | 阻力偏高，通行受阻 |
| 5 | cost > 21 | **險阻** | 紅 `#f56c6c` | 能量流嚴重受阻 |

**顯示格式**：`<標籤> (XX.XX)` — 標籤以對應色彩渲染；括號內精確值供硬核玩家參考。

- **二十主樞列表**：型 A（N000→主樞）顯示為「適性：暢流 (14.73)」。
- **觀測儀藍點**：型 B（主樞→藍）顯示為「─ 順通 (9.28)」；**型 E（藍環）** 以 `⟳→[承] 暢流 (5.32)` 標註環流至下一藍。
- **觀測儀綠點**：型 C（藍→綠）顯示為「─ 滯澀 (18.44)」；**型 D（綠環）** 以 `⟳→G02 暢流 (5.32)` 標註環流至下一綠。

**五類邊完整對應**：型 A（紅→橘）、型 B（橘→藍）、型 C（藍→綠）、型 D（綠同層環）、型 E（藍同層環）— 五類 760 條邊的 Cost 均已在星盤 UI 呈現。

**設計理由**：(1) 世界觀一致（能量流動語境）；(2) 色彩漸層一眼辨識偏科；(3) 保留精確值供比較；(4) 環流 Cost 以較小字體、半透明呈現，避免喧賓奪主。
