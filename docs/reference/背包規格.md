# 背包規格

> 頂欄「背包」按鈕 → 獨立彈窗。與人物彈窗（狀態/裝備/技能/星盤）分離：人物彈窗回答「我是誰」，背包彈窗管理「我帶了什麼」。
>
> 收斂自 [gemini chat_背包設定](gemini%20chat_背包設定.md) 與主管確認。對齊 [裝備分頁規格](裝備分頁規格.md)、[人物屬性彙整](人物屬性彙整.md) §一（體質：承傷、**負重**、耐性）、[經濟彙整](經濟彙整.md)。

---

## 一、核心機制：古典 MUD 負重制

- **無格子限制**：背包為 flat list，不做容器巢狀。
- **唯一限制為負重**：`最大負重 = 體質 × 10`。
- **超重懲罰**：超重時不可移動（第一版僅此懲罰）。
- **丟棄即掉落**：物品從背包移除，進入當前房間（待實作，第一版不做）。

---

## 二、物品分類

items 表以 `item_type` 區分：

| item_type | 說明 | 可堆疊 | 範例 |
|-----------|------|--------|------|
| `currency` | 鎂（實體貨幣，有面額，佔重量） | 是 | 鎂幣(1)、鎂鈔(10)、鎂錠(10000) |
| `equipment` | 可穿戴至 17 槽位 | 否 | 墨晶長劍、粗布短褂 |
| `token` | 詞元（入竅、鑲嵌、祭煉用） | 同種同屬性可堆疊 | 通用詞元 [石]、剝名所得 [狂] |
| `consumable` | 使用後消失 | 是 | 丹藥、食物 |
| `misc` | 其他（材料、任務物） | 是 | 獸皮、破碎器皿 |

---

## 三、堆疊規則

- **可堆疊**：`item_id` 相同且 items 表 `stackable = 1` → 合併為 `{item_id, qty}`。
- **不可堆疊**：裝備（每件可有獨立鑲嵌）、帶屬性的剝名詞元 → 每件獨立。

---

## 四、鎂（實體貨幣）

鎂是實體物品，佔重量，有面額。三種面額：

| item_id | 名稱 | 面額（鎂） | 單件重量 | stackable |
|---------|------|-----------|---------|-----------|
| `mg_coin` | 鎂幣 | 1 | 0.01 | 是 |
| `mg_note` | 鎂鈔 | 10 | 0.005 | 是 |
| `mg_ingot` | 鎂錠 | 10,000 | 1.0 | 是 |

items 表額外欄位 `denomination INTEGER` 記錄面額，供經濟系統換算。

第一版鎂仍以 entities 表 `magnesium` 欄位存放（整數，最小單位＝1 鎂），不進背包。待 V2 拆分為面額物品入背包。

---

## 五、UI 位置

頂欄第一個「保留」位 → 改為「背包」按鈕。點擊開啟獨立彈窗（不在人物彈窗內）。

### 5.1 彈窗內容

```
── 背包 ──────────────────
負重：12.5 / 100.0

  獸皮 ×8             (4.00)
  通用詞元 [石] ×12    (0.12)
  殘破的墨晶劍         (2.50)

（背包空空如也）← 無物品時顯示
──────────────────────────
```

- 頂部：負重摘要（當前 / 最大）。
- 列表：可捲動的純文字。每行顯示名稱、數量（堆疊時）、重量。
- 空背包顯示：`（背包空空如也）`。

## 六、後端資料結構

### 6.1 items 表擴充

| 欄位 | 類型 | 說明 |
|------|------|------|
| `weight` | REAL | 單件重量（預設 0） |
| `item_type` | TEXT | 物品分類（預設 'misc'） |
| `stackable` | INTEGER | 0=不可堆疊，1=可堆疊（預設 0） |

### 6.2 entities 表新增

| 欄位 | 類型 | 說明 |
|------|------|------|
| `inventory` | TEXT | JSON 陣列，每個元素 `{"item_id":"xxx","qty":1}`（預設 '[]'） |

### 6.3 負重計算

```
current_weight = Σ (item.weight × entry.qty)  for each entry in inventory
max_weight = entity.vit × 10
```

---

## 七、實作優先序

1. **第一版（骨架）**：頂欄按鈕 + 彈窗 UI + 後端回傳空背包。schema 預留。
2. **第二版（物品系統）**：物品掉落、拾取、丟棄、房間物品。
3. **第三版（穿脫互動）**：從背包穿上裝備、脫下裝備進背包。
4. **第四版（鎂實體化）**：鎂拆為面額物品，從 magnesium 欄位遷移至 inventory。
