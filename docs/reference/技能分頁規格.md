# 技能分頁規格（人物彈窗 UI）

> 人物點擊彈窗之【技能】分頁的 UI 結構與戰鬥底層邏輯。收斂自 [gemini chat_技能分頁](gemini%20chat_技能分頁.md)。對齊 [狀態與星盤分頁規格](狀態與星盤分頁規格.md) §一、[裝備分頁規格](裝備分頁規格.md)、[詞盤彙整](詞盤彙整.md)、[361拓撲系統規格](361拓撲系統規格.md)、[人物屬性彙整](人物屬性彙整.md)。

**核心原則**：

- **全機率 Tick 運算**：無主動施放。戰鬥為後端每 Tick 依權重擲骰自動演算，前端僅呈現富文字敘事 Log。
- **內定外動**：內盤（星盤）詞元煉化入體後極難置換，決定角色絕對基底；流派切換主要靠更換**裝備**與**運轉功法**來重組招式池。
- **田忌賽馬**：輸贏靠配置與策略，不靠操作。低傷招式若能在環境留下關鍵詞元引發連鎖，即為無價之寶。

## 一、戰鬥邏輯底層（後端預留）

### 1.1 廢除主被動區分

所有招式（無論來源）皆為「機率觸發」，由後端依權重每 Tick 擲骰決定。玩家不手動點擊施放。

### 1.2 實戰權重池（Combat Pool）與 SkillObject

系統在記憶體中維護角色的可用招式池。池中每個元素為一個**預編譯的 SkillObject**——語意判定僅在**煉化入竅 (Commit)** 或**功法習得**時執行一次，結果編譯為 SkillObject 存入池中；戰鬥時不重算語意，只讀 SkillObject。

**每 Tick 觸發公式**：

```
觸發權重 = SkillObject.基礎觸發率
         × 有效維度加權（體敏氣，依招式類型取對應維度）
         × 功法熟練度係數
         × 持物適配度（見 §1.6）
```

**傷害／效果公式**：

```
實際效果 = SkillObject.基礎效果 × 三軸係數
```

其中三軸係數依 [人物屬性彙整](人物屬性彙整.md) §四：

- **能階 (Amplitude)**：影響傷害規模（高能階 = 爆發力強）。
- **時脈 (Frequency)**：影響觸發節奏（高時脈 = 連擊傾向）。
- **相位 (Phase)**：影響連鎖反應的**散佈模式**（見 §1.4.1）。

同一招在不同念紋的人手裡效果截然不同——這是「萬人萬相」在戰鬥端的延伸。

### 1.2.1 資源檢定與反噬

每個 SkillObject 帶有資源消耗（`cost`：氣力、體力等）。Tick 擲骰抽中某招後，系統檢查角色當前資源是否足夠：

- **資源充足**：正常施放。
- **資源不足**：進入 A/B 雙軌判定——
  - **(B) 反噬（高機率）**：強制施放，不足的資源消耗以 **1:2** 比例直接扣除氣血（HP）。Log 產出紅色警告敘事（如「氣力乾涸，強行催動『霸皇斬』致氣血逆流」）。
  - **(A) 力竭（低機率）**：招式施放失敗（Fizzle），本 Tick 無動作。Log 產出灰色提示（如「後繼無力，攻勢化為烏有」）。

**設計意圖**：玩家無法在戰鬥中干預，但戰後可從 Log 中檢討——若反噬頻繁導致陣亡率高，迫使玩家重新配置招式池（降低高耗能招比例、補充回氣手段）。這是放置戰鬥的核心養成毒性。

> A/B 機率比例（如 B:A = 7:3）為框架預留，數值後定。

### 1.3 招式來源與三階共振

招式池的內容取決於角色的**造句共振階層**（見 [人物屬性彙整](人物屬性彙整.md) §五）與**功法自帶招式**：

| 來源 | 共振階層 | 進池條件 | 說明 |
|------|----------|----------|------|
| **功法自帶** | — | 習得即進池 | 來自習得之功法（劍法、拳術、指法、身法、心法等），附帶招式直接進入招式池。 |
| **內盤造句** | 一階（物理連通） | **不進池** | 詞元語意無關聯，僅提供竅穴基礎數值加成，不產出獨立招式。 |
| **內盤造句** | 二階（語意共振） | 進池 | 詞元具敘事連貫性（如「飛天梯雲縱」），系統判名產出動態機制／招式。 |
| **內盤造句** | 三階（純粹共振） | 進池 | 詞元屬性高度單一（如「火火炎燄轟」），極端效果招式，觸發率極低。 |
| **裝備造句** | 二階以上 | 進池 | 外盤（裝備）詞元造句共振，由系統自動萃取 2～4 字命名。見 [詞盤彙整](詞盤彙整.md) §五。 |

**關鍵**：一階共振僅提供數值加成（體敏氣），不產出招式、不進招式池。只有二階以上的語意共振才會編譯為 SkillObject 並進入 Combat Pool。

### 1.4 語境殘留（Context Residue）

戰鬥具有環境記憶。招式觸發後，其核心詞元殘留於戰場環境中（持續數 Tick）。若後續 Tick 出現相容詞元，系統自動引發**語意連鎖反應**（如 `[火]` + `[風]` = 爆燃）。

- 戰鬥不僅是比權重，更是文字版的化學沙盒。
- 低傷招式可作為「引信」，鋪設環境伏筆引爆高階連鎖。
- 語境推演供玩家戰前配置時參考。

#### 1.4.1 相位與連鎖散佈模式（框架）

連鎖反應觸發時，角色的**相位 (Phase)** 決定傷害如何散佈：

| 相位傾向 | 散佈模式 | 說明 |
|----------|----------|------|
| 秩序（Phase 趨近 +1.0） | 精準單體 | 傷害鎖定主目標，數值穩定（浮動小）。 |
| 中性（Phase 趨近 0） | 一般散佈 | 依預設規則散佈。 |
| 混沌（Phase 趨近 -1.0） | 房間 AOE | 傷害在敵方陣營內隨機彈射，數值大起大落。**絕不波及自身與友方**。 |

> 具體係數與散佈公式為框架預留，待戰鬥引擎實作時補齊。

### 1.5 功法星位共鳴（反向優化）

功法對星盤的刻印不限於產出招式。長期修練某功法，會對關聯之星位產生**反向優化**（降低阻力、提升詞元權重等）。功法類型以真實武學命名（劍法、拳術、指法、身法、心法……），不使用系統面泛稱。

### 1.6 無掛載上限：動態權重引擎

**不設功法掛載數量上限**。角色習得的所有武學永遠存在於招式池中，差別在於熟練度與當前情境。自然淘汰機制靠動態權重而非人為格數限制。

**動態權重的四個因子**：

| 因子 | 說明 |
|------|------|
| **熟練度係數** | 見 §1.7。勤練則權重高，荒廢則權重低。 |
| **持物適配度** | 當前持物（裝備 `hold_l` / `hold_r`）與招式所需器物的匹配程度。手持長劍時，劍系招式適配度極高，空手系（拳、擒拿）降至極低或歸零；反之亦然。空手時所有徒手系招式適配度最大化。 |
| **心法共鳴** | 當前運轉的心法對相容技法提供全域觸發率加成。頂級心法共鳴範圍廣，平民心法共鳴範圍窄。 |
| **有效維度** | 體敏氣依招式類型取對應維度加權（同 §1.2）。 |

**最終觸發權重**（完整公式）：

```
觸發權重 = SkillObject.基礎觸發率
         × 熟練度係數
         × 持物適配度
         × 心法共鳴係數
         × 有效維度加權
```

**舉手投足皆武器**：若一個角色練了足技、關節技、擒拿術、手法、肘技，且熟練度皆高，則空手狀態下這些招式全數高權重進池——舉手投足皆武器。但當他拔出長劍，劍系招式浮現，徒手系自然沉底。

**頂級功法 = 複合矩陣**：一本頂級功法（如《天極霸皇劍》）不只是劍法，內含劍陣、步法、指法、心法等多個子類。其 `martial_matrix` 定義了各子類的權重占比。即使劍被打飛，該功法中的步法與指法招式仍可觸發——手中無劍，心中有劍。

**心法的特殊地位**：心法為能量流轉根基，同時僅運轉一套（切換心法 = 切換全域共鳴底色）。外功不限數量。

> 心法掛載上限：**1 套**。外功掛載上限：**無限制**。

### 1.7 熟練度

功法熟練度影響招式觸發率與星位共鳴強度。

| 等級 | 名稱 | 觸發率係數 | 說明 |
|------|------|-----------|------|
| 0 | **未學** | — | 未習得，不進池。 |
| 1 | **初窺** | ×0.6 | 剛習得，招式觸發率低。 |
| 2 | **小成** | ×0.8 | 基本掌握。 |
| 3 | **大成** | ×1.0 | 完整發揮基礎觸發率。 |
| 4 | **圓滿** | ×1.2 | 爐火純青，觸發率溢出。 |

**提升方式**（框架，數值後定）：累計使用次數（戰鬥中該功法招式被觸發的次數達到門檻即升級）。

## 二、【技能】分頁 UI 結構

分頁劃分為三個層次分明的純文字區塊。第一版可先做骨架佔位。

### 區塊 1：運轉功法

上半顯示**當前心法**（僅一套，決定全域共鳴底色），下半列出角色**所有已習得的外功**，依熟練度排序。

```
【 運轉功法 】
▣ 心法：《混元吐納術》 (熟練：圓滿)  [大循環]
  └ 星盤滋養：為全星盤提供穩定氣脈底噪。
────────
◆ 劍法：《天極霸皇劍》 (熟練：小成)  [大循環]
  ├ 矩陣：劍陣 50% ｜ 步法 20% ｜ 指法 15% ｜ 心法 15%
  └ 星位共鳴：引動 [天極] 主樞，提升該星系攻擊性詞元威能。
◆ 拳術：《莊稼漢拳腳》 (熟練：初窺)  [微循環]
  └ (無星位共鳴)
◆ 擒拿：《江湖扣鎖手》 (熟練：大成)  [微循環]
  └ (無星位共鳴)
◆ 足技：《旋風連環腿》 (熟練：小成)  [微循環]
  └ (無星位共鳴)
```

- **▣ 心法**：獨立置頂，同時僅運轉一套。切換心法 = 切換全域共鳴底色，招式池瞬間重算。
- **◆ 外功**：不限數量，全部列出。熟練度高者排前。
- **`[微循環]`**：僅走綠點表層的功法（見 [361拓撲系統規格](361拓撲系統規格.md) §6.2），無星位共鳴。
- **`[大循環]`**：路徑經橘→藍→綠的功法，具星位共鳴效果。
- **矩陣**：頂級功法展開其包含的子類權重占比（僅複合型功法顯示）。

### 區塊 2：實戰招式池

列出系統盤點後，當前配置（功法 + 內盤造句 + 裝備造句）在戰鬥中有機率觸發的所有招式。僅二階以上共振與功法自帶招式會出現在此。

```
【 實戰招式池 】
⚔ 霸皇斬 ｜ 觸發率：極高 (受 劍法熟練 增幅)
  ├ 來源：劍法《天極霸皇劍》
  └ 描述：牽動天極星位動能，造成巨量物理傷害。

🔥 焦土 ｜ 觸發率：極低 (三階純粹共振)
  ├ 來源：內盤語意共振 (火/炎/轟)
  └ 描述：系統判名機制，觸發絕對真實傷害。
```

### 區塊 3：語境推演

根據招式池推演玩家在實戰中最可能於環境留下的詞元變數，供戰術配置參考。每次更換功法或裝備時重算。

```
【 語境推演 】
◆ 環境干涉傾角：
  └ 高頻殘留：[ 鋒 ] (占比 45%) ｜ [ 火 ] (占比 30%)
◆ 預期連鎖反應：
  [ 熔甲 ] → 若環境同時存在 [火] 與 [鋒]，高機率瓦解敵方基礎防禦。
```

## 三、後端資料結構預留

### 3.1 功法結構（SkillManual）

| 欄位 | 類型 | 說明 |
|------|------|------|
| `id` | TEXT | 功法唯一 ID |
| `name` | TEXT | 功法名稱（如「天極霸皇劍」） |
| `slot_type` | TEXT | `inner`（心法）或 `outer`（外功） |
| `martial_matrix` | TEXT/JSON | 武學子類權重矩陣（見下）。簡單功法可為單一類型。 |
| `circulation` | TEXT | `micro`（微循環）或 `macro`（大循環） |
| `path_nodes` | TEXT/JSON | 功法路徑之節點 ID 序列（路徑定義） |
| `resonating_nodes` | TEXT/JSON | 此功法關聯的星位節點 ID 陣列（共鳴對象） |
| `resonance_effect` | TEXT | 對星盤的優化效果描述 |
| `moves` | TEXT/JSON | 附帶招式清單（含基礎觸發權重、`cost`、`residue_tokens`、`required_weapon_type`） |

**`martial_matrix` 範例**：

```json
// 頂級複合功法
{"劍法": 0.50, "步法": 0.20, "指法": 0.15, "心法": 0.15}

// 平民單一功法
{"拳術": 1.0}
```

矩陣中的子類權重影響該功法下各招式的**持物適配度計算**——劍被打飛後，劍法子類歸零，但步法、指法子類仍可觸發。

### 3.2 SkillObject 結構

招式池中的原子單位。煉化入竅或習得功法時編譯產出，戰鬥時僅讀取。

| 欄位 | 類型 | 說明 |
|------|------|------|
| `id` | TEXT | 招式唯一 ID |
| `name` | TEXT | 招式名稱（功法自帶為固定名、系統判名為自動萃取 2～4 字） |
| `source_type` | TEXT | `manual`（功法自帶）/ `inner_resonance`（內盤造句）/ `outer_resonance`（裝備造句） |
| `source_id` | TEXT | 來源功法 ID 或造句 ID |
| `resonance_tier` | INT | 共振階層：2（語意共振）或 3（純粹共振）；功法自帶填 0 |
| `sub_type` | TEXT | 招式所屬子類（劍法、拳術、步法、指法……），用於持物適配度計算 |
| `required_weapon_type` | TEXT/NULL | 需要的持物類型（如 `"鋒刃"`）；null 表示徒手可用 |
| `base_trigger_rate` | FLOAT | 基礎觸發率（0.0～1.0） |
| `cost` | TEXT/JSON | 資源消耗（如 `{"氣力": 50}` 或 `{"體力": 20}`），用於 §1.2.1 資源檢定 |
| `base_effect` | TEXT/JSON | 基礎效果結構（傷害值、效果類型、影響維度等） |
| `residue_tokens` | TEXT/JSON | 觸發後殘留於環境的詞元清單（如 `["火","鋒"]`） |
| `description` | TEXT | 招式敘事描述 |

### 3.3 戰鬥 Log 原子結構

後端每 Tick 產出的單條 Log，為前端渲染富文字的唯一資料契約。支援單體與全房 AOE。

| 欄位 | 類型 | 說明 |
|------|------|------|
| `tick` | INT | 當前 Tick 序號 |
| `actor` | TEXT | 施招者 entity_id |
| `targets` | JSON | 受擊者 entity_id **陣列**（支援單體與 AOE：`["npc_001"]` 或 `["npc_001","npc_002","npc_003"]`） |
| `move_id` | TEXT | 觸發的 SkillObject ID |
| `move_name` | TEXT | 招式名稱 |
| `source` | TEXT | 來源（功法名或造句描述） |
| `result` | TEXT | 結果（命中／閃避／格擋／反噬／力竭……） |
| `damage` | INT | 造成傷害（0 表示未命中或非傷害招；反噬時為自傷值） |
| `resource_cost` | JSON/NULL | 實際消耗的資源（如 `{"氣力": 50}` 或反噬時 `{"氣血": 100}`） |
| `residue_added` | JSON | 本次新增至環境的殘留詞元 |
| `chain_triggered` | TEXT/NULL | 若觸發連鎖反應，填連鎖名稱；否則 null |
| `narrative` | TEXT | 富文字敘事（前端直接渲染） |

### 3.4 戰鬥引擎預留（Combat Engine）

每 Tick 執行順序：

1. **擲骰選招**：依完整觸發權重公式（§1.2 + §1.6）從 Combat Pool 加權抽取。
2. **資源檢定**：檢查 `cost` 是否滿足（§1.2.1），不足則進入 A/B 雙軌判定。
3. **效果計算**：`基礎效果 × 三軸係數`，相位決定散佈模式（§1.4.1）。
4. **語境更新**：將 `residue_tokens` 加入環境殘留清單，檢查連鎖觸發條件。
5. **產出 Log**：填充 §3.3 原子結構，推送前端。

- **三軸係數**：SkillObject 基礎效果 × 角色三軸係數 = 實際效果。同一招在不同念紋的角色手裡傷害與節奏不同。
- **NPC 對等**：NPC 互打邏輯與玩家打怪完全相同（同一套 Pool 對撞），支撐萬名 NPC 背景運算。
- **語境殘留**：維護戰場環境的詞元殘留清單（每條帶 TTL），每 Tick 檢查連鎖觸發條件。
- **房間制 AOE**：混沌相位連鎖或特定群攻招式可命中同房多個目標（`targets` 陣列），致敬 MUD 全房叫殺。

### 3.5 裝備結構對接

更換 `equipment_slots`（17 個槽位）會即時改變**持物適配度**，招式池中各招的觸發權重瞬間重算。拔劍→劍系招式浮現、徒手系沉底；卸劍→徒手系浮現、劍系沉底。內盤詞元不動、外盤（裝備 + 功法）可換，即「內定外動」。
