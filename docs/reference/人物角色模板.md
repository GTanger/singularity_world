# 人物角色模板

> 玩家與 NPC 共用同一套實體結構（玩家＝NPC）。對齊 [人物屬性彙整](人物屬性彙整.md)、[詞盤彙整](詞盤彙整.md)、[決策 002](decisions/002_sockets_plugs_semantic.md)、[決策 005](decisions/005_room_cell_view_scope.md)（一格＝一房＝視線所及）、[第一版可做清單](../第一版可做清單.md) §1.5。

---

## 一、識別與類型

| 欄位 | 類型 | 說明 |
| --- | --- | --- |
| **id** | 唯一識別 | 實體 ID；玩家與 NPC 皆唯一。 |
| **kind** | 枚舉或字串 | `player` \| `npc`；語義上皆為 Agent，僅區分是否由真人操作。 |
| **display_char** | 單字（中文） | 圓內一字，用於地圖與列表顯示；可依身份／門派／設定分配。 |

## 三、屬性（A+B：基礎＋詞盤驅動）

### 3.1 先天：SoulSeed（361＋三軸之唯一來源）

| 欄位 | 類型 | 說明 |
| --- | --- | --- |
| **soul_seed** | int64 | 創角時由後端產生並**必存**，僅屬於該角色。由此 seed 決定性展開：**三軸信號光譜**（Amplitude、Frequency、Phase）與 **760 條拓撲邊權**（節點間阻力／難易度），見 [人物屬性彙整](人物屬性彙整.md) §2.0、[361拓撲系統規格](361拓撲系統規格.md) §6.1。 |

實作可只存 `soul_seed`、載入時依固定 RNG 順序重算三軸與邊權；或創角時展開並快取三軸與 760 Cost 以加速載入。**基礎體敏氣**由三軸映射產出（人物屬性彙整 §二）；舊版「出身×念紋」已收斂為 SoulSeed＋三軸。

### 3.2 基礎有效屬性（可快取）

| 欄位 | 類型 | 說明 |
| --- | --- | --- |
| **體質** | 數值 | 承傷、負重、耐性；由 **三軸映射** 產出（§3.1、人物屬性彙整 §二），不隨等級成長。 |
| **氣脈** | 數值 | 續戰、技能／行動消耗、回復。 |
| **靈敏** | 數值 | 命中、閃避、行動序。 |

與 §3.1 一致：基礎體敏氣唯一來源為 SoulSeed 展開之三軸；無出身表、無四型念紋。第一版可先固定一組或僅實作三軸映射公式。

### 3.3 竅穴與詞元（內盤）與星盤貫通狀態

| 欄位 | 類型 | 說明 |
| --- | --- | --- |
| **activated_nodes** | 列表（JSON 或 TEXT） | 已貫通之 361 節點 ID 清單；預設 `["N000"]`（生之奇點預設點亮）。前端星盤渲染時，不在清單內之節點顯示為「未貫通」或「迷霧」。第一版不實作打通流程，僅預留此欄位。見 [狀態與星盤分頁規格](狀態與星盤分頁規格.md) §五.3。 |
| **竅穴打通狀態** | 列表或位元 | 哪些竅穴已打通→對應普通加成（體質／氣脈／靈敏）；查 [詞盤彙整](詞盤彙整.md) §9.1。可與 activated_nodes 對接。 |
| **詞元入竅** | 列表 | 各竅穴填入之詞元 ID 或屬性→特殊加成；查 [詞盤彙整](詞盤彙整.md) §9.2。 |
| **連線** | 結構或 ID 列表（選填） | 竅穴連線模板→額外加成；查 [詞盤彙整](詞盤彙整.md) §9.3。 |

第一版可簡化：不做竅穴／詞元，僅用 3.2 基礎有效屬性；**activated_nodes** 預留並預設 `["N000"]`，主樞適性先寫死或佔位。

### 3.4 裝備

| 欄位 | 類型 | 說明 |
| --- | --- | --- |
| **裝備槽** | 鍵值或列表 | 全身多部位裝備；每槽可存裝備 ID 或 null。裝備提供「工具維度」加成，與有效屬性疊加。 |

第一版可簡化：少量槽位（如武器、防具各一）或僅「裝備 ID 列表」。

## 七、觀測與坍縮（決策 004、005）

| 欄位 | 類型 | 說明 |
| --- | --- | --- |
| **last_observed_at** | 時間或 tick | 上次被任一玩家觀測的時間；用於惰性求值與延遲坍縮。 |
| **observed_by** | 玩家 ID 列表或 null | 當前在誰的視野內；視野內才即時模擬。**視野＝與玩家同房**（決策 005：一格＝一房＝視線所及）。 |

## 八、彈窗分頁顯示（狀態／裝備／技能／星盤）

點擊自己或 NPC 打開彈窗時，四個分頁為：**狀態**｜**裝備**｜**技能**｜**星盤**。**玩家＝NPC**：同一套實體結構，差異在「誰在看」決定**可見範圍**。UI 結構與用語依 [狀態與星盤分頁規格](狀態與星盤分頁規格.md) 實作；以下為欄位與後端對應摘要。

### 8.1 【狀態】分頁 — 共通（自己與看別人都可顯示）

| 區塊 | 項目 | 說明與備註 |
|------|------|------------|
| **命途** | 稱謂／身份 | 第一版可寫死「無名之輩」或由系統給初始字串 |
| **維度** | 有效體質、有效氣脈、有效靈敏 | 外顯三維，見人物屬性彙整 §一 |
| **四相** | 氣血、內力、精神、體力 | 四項資源統稱「四相」；顯示 當前/最大；滿條＝該屬性最大值，條寬＝當前/最大×100％ |
| **持有** | 鎂 | 看他人時可依設計隱藏或顯示為「？」 |

**四項資源最大值公式**（由體質 V、氣脈 Q、靈敏 D 計算；**無條件進位至整數**；當前值第一版＝最大值）：  
- **氣血最大值** ＝ ⌈((0.7×V)+(0.3×Q)) × ((0.05×D)+1) × V⌉  
- **內力最大值** ＝ ⌈((0.7×Q)+(0.3×V)) × ((0.05×D)+1) × Q⌉  
- **精神最大值** ＝ ⌈((0.6×D)+(0.4×Q)) × ((0.05×V)+1) × D⌉  
- **體力最大值** ＝ ⌈((0.5×V)+(0.4×Q)+(0.3×D)) × ((V+Q+D)/3)⌉  

以上為**外顯、可被觀察**的狀態；後端依人物屬性彙整公式算出有效體敏氣與四項資源，前端僅展示。

### 8.2 【狀態】分頁 — 僅自己（點自己時多出）

| 區塊 | 說明 | 備註 |
|------|------|------|
| **本源** | 三軸語感之**一句話**描述 | 由 Amplitude／Frequency／Phase 轉為語感形容詞拼成一句；**不直接顯示**能階／時脈／相位數字。見狀態與星盤分頁規格 §二。 |
| **基礎體質／氣脈／靈敏**（可選） | 由三軸映射產出的先天值 | 讓玩家區分「先天」與「有效（含竅穴／詞元／裝備）」 |
| **星盤／內視** | 拓撲貫通率、主樞適性、觀測儀樹狀等 | 僅自己、且需執行過內視；詳見 **【星盤】分頁**（§8.5）與 361 拓撲「經脈迷霧」 |

### 8.3 看 NPC／他人時

- 狀態分頁顯示 **§8.1 共通** 內容；**鎂**是否顯示由專案定。
- **不顯示**：本源、基礎體敏氣、星盤／內視（皆為內在，不對外暴露）。

### 8.4 【裝備】與【技能】分頁

- **裝備**：留白，待物品系統實裝。
- **技能**：詞元迴路／造句成果；留白待語意判定系統實裝。

### 8.5 【星盤】分頁（原「經脈」）

- 第四分頁命名為 **星盤**，對應 361 竅穴網路之層級式呈現（生之奇點→二十主樞→五藍→十二綠）。
- 內容：宏觀貫通率與 N000 生之奇點（已點亮）、二十主樞目錄（摺疊）、觀測儀樹狀（展開後綁定 Topology Cost）。
- 詳見 [狀態與星盤分頁規格](狀態與星盤分頁規格.md) §三、[361拓撲系統規格](361拓撲系統規格.md)。

### 8.6 第一版簡化

- 狀態分頁至少顯示：**命途、維度（體質／氣脈／靈敏）、四相（氣血／內力／精神／體力 當前/最大）、持有（鎂）**；本源可第一版寫死一句或暫不顯示。
- 星盤分頁可先做層級骨架與 Cost 綁定，打通／煉化按鈕可為空殼或 log。

## 九、第一版最小模板（建議）

第一版可收斂為以下欄位，其餘留後續；狀態分頁顯示見 §八。

- **id, kind, display_char**
- **x, y, move_state, target_x, target_y, walk_or_run**（移動用）
- **體質, 氣脈, 靈敏**（可固定一組或依念紋類型查表）
- **sockets**（最小集）
- **鎂**
- **last_observed_at**（NPC 用；玩家可省略或填 null）

出身／念紋／竅穴／詞元／裝備／名稱可第一版省略或固定，待詞盤與屬性完整接上再擴充。
