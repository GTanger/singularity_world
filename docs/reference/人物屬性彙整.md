# 人物屬性彙整（奇點世界）

> 人物屬性**以體質、氣脈、靈敏三維為對外介面**，本彙整說明此三維如何由新版底層（SoulSeed、361 拓撲、三軸光譜、詞元造句）產出。修訂來源：與 Gemini 討論見 [人物屬性gemini chat.md](人物屬性gemini%20chat.md)。與 [詞盤彙整](詞盤彙整.md)、[361拓撲系統規格](361拓撲系統規格.md) 對接；鎂產消見 [經濟彙整](經濟彙整.md)。

**核心哲學**：有限框架中的無限可能。

**外顯與內在**：**體質、氣脈、靈敏** 為角色最直觀的**外顯**三維——對戰鬥與生活系統暴露的有效屬性僅此三項。**內在**為 361 節點（含唯一 **生之奇點** N000 ＋ 其餘 **360 節點** 收斂為體／氣／敏 三系）與 **三軸信號光譜**；內在產出並驅動外顯體敏氣，裝備為可剝離之外在工具。

**前端 UI 用語對照**（人物彈窗【狀態】與【星盤】分頁）：本彙整之「體質、氣脈、靈敏」在 UI 區塊稱 **【維度】**；氣血、內力、精神、體力四項資源在 UI 統稱 **【四相】**；三軸信號光譜之敘事化一句話在 UI 區塊稱 **【本源】**（不直接顯示能階／時脈／相位數字）；361 竅穴網路在 UI 第四分頁命名 **【星盤】**。細部結構見 [狀態與星盤分頁規格](狀態與星盤分頁規格.md)。

## 一、外顯三維：體質、氣脈、靈敏

角色對戰鬥與生活系統暴露的**有效屬性**僅此三項，後端唯一真理依此結算。

| 維度 | 說明 | 主要影響 |
|------|------|----------|
| **體質** | 承傷、負重、耐性 | 減傷、霸體、生存 |
| **氣脈** | 續戰、技能／行動消耗與回復 | 續戰力、詠唱、回氣 |
| **靈敏** | 命中、閃避、行動序 | 命中、閃避、先手 |

**有效屬性公式**（每維相同結構）：

```
有效體質 = 基礎體質 + Σ 竅穴普通加成(體) + Σ 詞元／連線加成(體) + 裝備加成(體)
有效氣脈 = 基礎氣脈 + Σ 竅穴普通加成(氣) + Σ 詞元／連線加成(氣) + 裝備加成(氣)
有效靈敏 = 基礎靈敏 + Σ 竅穴普通加成(敏) + Σ 詞元／連線加成(敏) + 裝備加成(敏)
```

- **基礎體質／氣脈／靈敏**：由先天 **三軸信號光譜（念紋）** 線性映射產出，不隨等級膨脹，無出身表。
- **竅穴普通加成**：打通竅穴依對照表產出體／氣／敏加成；與 361 拓撲對接時，以「節點／區」對應體竅／氣竅／敏竅（見詞盤彙整 §9.1、§9.5）。
- **詞元／連線加成**：詞元入竅與連線依詞元屬性與語意判定產出特殊加成或技能；見 §五、詞盤彙整 §9.2／§9.3。
- **裝備加成**：外在詞盤（附魔／強化）提供之體／氣／敏維度，與內在相加後進入戰鬥。

以下各節說明**內在**（SoulSeed、361 拓撲、三軸、造句）如何產出**外顯**的基礎與竅穴／詞元／連線加成。

## 二、先天：SoulSeed 與「基礎體敏氣」的產出

### 2.0 創角時後端必存：僅屬於該角色的 361＋三軸串值

**角色誕生當下，後端必須寫入並永久保存該角色專屬的一筆種子**，由此唯一決定該角色的「三軸信號光譜」與「760 條節點間連線的阻力（難易／長度）值」：

| 紀錄內容 | 說明 |
|----------|------|
| **soul_seed** | 型別 `int64`，**必存**。創角時由後端產生（如時間戳＋實體 ID 之雜湊、或加密安全亂數），寫入實體表或專用表，此後不變。 |
| **三軸值** | Amplitude、Frequency、Phase 三數。由同一 SoulSeed 之前 **3 次 RNG** 決定性產出；**邏輯上**已隨 soul_seed 唯一確定。實作可**只存 seed、載入時重算**，或**創角時算一次並快取**（見下表）。 |
| **760 條邊權 (Cost)** | 節點代碼之間每條連線的阻力／難易度。由同一 SoulSeed 之**第 4～763 次 RNG** 依 [361拓撲系統規格](361拓撲系統規格.md) §6.1 決定性產出，總和恆為 CONST_TOTAL_TOPOLOGY_COST（10000）。**邏輯上**已隨 soul_seed 唯一確定；實作可只存 seed 載入時展開，或創角時展開並快取。 |

**實作策略二選一**：

- **(1) 只存 soul_seed**：資料庫僅一欄 `soul_seed`。角色載入時以該 seed 初始化 RNG，依**固定順序**（前 3 次→三軸，第 4～763 次→760 條邊權）展開，得到三軸與 760 個 Cost；不另存展開值。優點：儲存最小、無不一致風險。
- **(2) 創角時展開並快取**：創角時依上順序算出一組三軸與 760 Cost，寫入 `entity_soul` 或 `topology_edges` 等表；載入時直接讀取，免重算。適用於需頻繁讀取路徑成本、希望減少每次載入的 RNG 計算時。

**結論**：每個角色誕生時，後端**至少**會紀錄僅屬於該角色的 **soul_seed**；節點代碼之間的隨機長度（難易／阻力）值及三軸值，皆由該 seed **決定性生成**，可視為「已紀錄」——不論採用 (1) 或 (2)，邏輯上皆為「該角色專屬的 361＋三軸串值已固定並可重現」。

---

為效能與防作弊，資料庫**至少儲存**一筆 `soul_seed`（型別 `int64`）；角色載入時由此 Seed 決定性展開（或直接讀取已快取之展開值）：

1. **三軸信號光譜**（前 3 次 RNG）：能階 Amplitude [0.1, 3.0]、時脈 Frequency [0.5, 2.0]、相位 Phase [-1.0, 1.0]，用於與詞元互動之係數，並**映射為基礎體質、基礎氣脈、基礎靈敏**。
2. **361 拓撲邊權**（第 4～763 次 RNG，歸一化）：每條連線之阻力 Cost 由 SoulSeed 決定性生成，**全圖 Cost 總和恆等於常數 C**（10000），故無絕對廢柴或天才、只有偏科。演算法與邊序見 [361拓撲系統規格](361拓撲系統規格.md) §6.1。

**基礎體敏氣由三軸產出（建議實作）**：  
由三軸線性映射或常數表產出少量基礎值，例如  
`基礎體質 = base × (1 + k_amp × (Amplitude - 1))`、  
`基礎氣脈 = base × (1 + k_freq × (Frequency - 1))`、  
`基礎靈敏 = base × (1 + k_phase × Phase)`  
（軸與維度對應可依平衡調整）。單一來源、無舊出身／四型殘留。

**經脈迷霧**：創角時不顯示拓撲與阻力；玩家透過 `[內視]`（消耗氣脈／鎂）逐步探明，避免刷首抽。  
完整演算法見 [人物屬性gemini chat.md](人物屬性gemini%20chat.md) Turn 45–46。

## 三、先天硬體：361 竅穴拓撲（360 收斂為體／氣／敏 三系）

人體為 361 節點之拓撲網格：**唯一生之奇點 N000** 為能量起點；其餘 **360 節點**（二十主樞、五常邏輯閘、十二微型狀態）收斂為 **體系、氣系、敏系**，與外顯之**體質、氣脈、靈敏**一一對應。此結構為**內在**，決定路徑、阻力、可打通的節點，從而決定「對外顯體敏氣能貢獻多少竅穴普通加成與造句槽位」。

- **打通**：依功法藍圖在拓撲上點亮節點；若主徑阻力過高可依**同層橫向跳線**繞路（自創功法）。打通之節點依其**系屬**產出對應之**體質／氣脈／靈敏普通加成**（詞盤彙整 §9.1）。**系屬對照**：每個節點 N001～N360 之體／氣／敏歸屬已明定於 [361拓撲系統規格](361拓撲系統規格.md) §2.2（二十主樞系屬表）、§2.3（體系／氣系／敏系 節點代碼一覽）；主樞所轄 5 藍、12 綠與該主樞同系屬，後端可依 §2.1 公式或 §2.3 表判定任意 Nxxx 之系屬。
- **第一版**：可採「雙軌並存」——361 負責路徑與造句；普通加成依 361 規格 §2.2／§2.3 查系屬後加對應之體／氣／敏。

分區與代碼見 [361拓撲系統規格](361拓撲系統規格.md)；討論與示意見 [人物屬性gemini chat](人物屬性gemini%20chat.md) Turn 46–56。

## 四、先天軟體：三軸信號光譜（念紋）

念紋為三軸浮點數，表示靈魂在詞盤網路中的語意向量／腦波指紋；除產出**基礎體敏氣**外，與詞元／行為互動時決定消耗、變異與 Log 敘事。

| 軸向 | 系統名 | 區間 | 語感（低←→高） |
|------|--------|------|------------------|
| **能階係數** | Amplitude | [0.1, 3.0] | 幽微／綿長 ←→ 霸道／爆發 |
| **時脈帶寬** | Frequency | [0.5, 2.0] | 渾厚／重劍無鋒 ←→ 洞察／千絲萬縷 |
| **順逆向量** | Phase | [-1.0, 1.0] | 順流／秩序 ←→ 逆流／混沌 |

- 與詞元互動時，後端以三軸與詞元標準向量之內積（或等價運算）決定消耗與變異；反相極高者用秩序詞元易走火入魔，用毒／破壞系則可能觸發變異效果。
- 戰鬥時技能效果讀取編譯後之 SkillObject，再乘上三軸係數（如能階影響傷害規模、時脈影響節奏等）。

## 五、後天：詞元入竅與造句（對體敏氣與技能的貢獻）

功法規定**路徑順序**；玩家在路徑節點煉化**詞元**。能量依序流過詞元時，系統做**語意判定**，產出對體敏氣的**詞元／連線加成**與可選的**技能**（SkillObject）。

| 階層 | 名稱 | 條件（概念） | 對體敏氣與技能的貢獻 |
|------|------|--------------|------------------------|
| **第一階** | 物理連通（囫圇吞棗） | 詞元語意無關聯、不成句 | 僅各竅穴之**基礎數值加成**（依詞元屬性對應體／氣／敏，見詞盤 §9.2），無額外技能。 |
| **第二階** | 語意共振（邏輯成句） | 詞元具敘事連貫性（如：飛天梯雲縱） | 加成之外，解鎖**動態機制／技能**（如強制閃避、無視地形）。 |
| **第三階** | 極端疊加（純粹共振） | 詞元屬性高度單一（如：火火炎燄轟） | 極端數值爆發、真實傷害等，反映在傷害或穿透類效果。 |

- **繞路與造句**：因體內阻力繞路時節點數改變，須重新湊詞元造句；新句可觸發不同語意與變異技能。
- **效能**：語意判定僅在 **[煉化入竅 (Commit)]** 時執行一次，結果編譯為 SkillObject；戰鬥只讀 SkillObject × 三軸係數。
- **第一版語意實作建議**：關鍵詞＋白名單句型；成句＝詞元序列匹配認可句型，純粹＝同屬性連續 N 個；未匹配則歸第一階。

詞元鑲嵌在不同綠點（如【蓄】vs【釋】）會改變效果類型；細則見 [詞盤彙整](詞盤彙整.md)。

## 七、邏輯閉環與第一版簡化

**閉環**：SoulSeed → 三軸（→ 基礎體敏氣）＋拓撲阻力 → 功法路徑／打通／繞路 → 竅穴普通加成(體氣敏) ＋ 詞元入竅與造句 → 詞元／連線加成(體氣敏) ＋ SkillObject → 有效體敏氣 ＋ 裝備 → 戰鬥 f(有效體敏氣, 技能, 裝備)。與詞盤、經濟、玩家＝NPC 一致。

**閉環自檢（連結順暢度）**：

| 環節 | 上游 | 下游 | 狀態 |
|------|------|------|------|
| 創角 | — | soul_seed 必存 | ✅ 人物屬性 §2.0、schema、角色模板 §3.1 |
| 三軸＋邊權 | soul_seed、固定 RNG 順序 | 基礎體敏氣、760 Cost | ✅ §二、361 規格 §6.1、soulseed_demo |
| 基礎體敏氣 | 三軸線性映射 | 有效屬性公式第一項 | ✅ §一、§二 |
| 打通／路徑 | 功法藍圖、拓撲 Cost | 竅穴普通加成 | ⚠️ 功法藍圖（路徑定義）來源見下方斷點 |
| 系屬→加成 | 361 §2.2／§2.3 | 體／氣／敏 普通加成 | ✅ 人物屬性 §三、詞盤 §9.1／§9.5 |
| 詞元／造句 | 煉化、語意判定 | 詞元／連線加成、SkillObject | ✅ §五、詞盤 §9.2／§9.3／§9.4 |
| 有效體敏氣 | 基礎＋竅穴＋詞元／連線＋裝備 | 戰鬥、經濟消耗 | ✅ §一、§六、詞盤 §9.5 |

**已知斷點（第一版可簡化或後補）**：  
① **功法藍圖**：路徑（哪些節點依序打通）由「功法」定義，規格未單獨成檔；第一版可用靜態功法表（每條功法 ID 對應一組節點序列），或僅做單點打通不強制路徑。  
② **內視**：揭露拓撲／阻力的時機與消耗（氣脈／鎂）見 §二「經脈迷霧」與第一版簡化④；細則可於第一版可做清單補。  
③ **SkillObject**：造句→系統判定通過後產出「可被戰鬥讀取的技能結構」；該結構欄位與 ID 可於戰鬥／詞盤細規補齊。

**第一版可簡化**：  
① 實作 SoulSeed 展開三軸並映射為基礎體敏氣；拓撲可先做少量邊權或簡化圖。  
② 竅穴普通加成用詞盤 §9.1 體竅／氣竅／敏竅，打通與 361 以「區／組」對照即可。  
③ 語意判定用關鍵詞＋白名單句型，Commit 時產出 SkillObject。  
④ 內視可設計為分段解鎖（每消耗氣脈／鎂揭曉一段阻力），並給模糊反饋（如某主樞適性高／低）。

**已廢除**：四型念紋、出身×念紋加權。念紋＝三軸光譜；基礎體敏氣＝三軸映射，單一來源。

完整 361 表、二十主樞、五藍十二綠、實作指導見 [361拓撲系統規格](361拓撲系統規格.md)、[人物屬性gemini chat](人物屬性gemini%20chat.md)。
