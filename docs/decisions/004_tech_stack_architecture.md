# 決策 004：技術棧與架構

**狀態**：已決斷  
**依據**：主管決斷

---

## 1. 決策內容

### 1.1 伺服器：Go

- **遊戲主迴圈**、**NPC 即時行為**（僅視野內）
- **量子坍縮系統**：惰性求值 + 事件日誌
- **經濟／政治／生態引擎**：獨立 goroutine，各自 tick rate
- **WebSocket 伺服器**，處理最多 **10 名玩家**連線
- **SQLite** 持久化

### 1.2 前端：網頁（HTML + CSS + JavaScript）

- **WebSocket** 連接伺服器
- **Canvas** 畫 2D 俯視地圖 + 符號移動
- **HTML 原生富文字**處理詞盤系統、對話、敘事
- **不需安裝**，開瀏覽器就玩

### 1.3 資料庫：SQLite

- NPC 狀態、事件日誌、經濟數據、玩家存檔
- 十人規模，SQLite 綽綽有餘

---

## 2. 架構核心原則

- **90,000 NPC 平時不跑**：只存狀態 + 上次觀測時間。
- **觀測發生時**：從事件日誌回推算出當前狀態（**延遲坍縮**）。
- **視野內 NPC** 才進入即時模擬。
- **全域系統**（經濟、政治、生態）持續在背景跑，產出事件流。

---

## 3. 與專案設計對齊

| 設計要點 | 對齊方式 |
| --- | --- |
| 後端唯一真理 | Go 伺服器為唯一邏輯與狀態來源；前端僅顯示與發送操作 |
| 萬名異步 NPC、觀測與坍縮 | 九萬 NPC 惰性存狀態；觀測時由事件日誌回推（坍縮）；視野內才即時模擬 |
| 同服最多約十人 | WebSocket 最多 10 名玩家連線，與關鍵字一致 |
| 手機、無上架 | 網頁前端，開瀏覽器即玩，無需安裝與上架 |
| 城鎮即大地圖、以字符示意 | Canvas 2D 俯視 + 符號移動；詞盤／對話／敘事用 HTML 富文字 |
