# 決策：戰鬥規則統一（玩家 vs NPC / NPC vs NPC / 玩家 vs 玩家）

**狀態**：已決斷  
**依據**：設計關鍵字（戰鬥全自動、後端唯一數據真理、玩家＝NPC）、主管決斷

---

## 1. 決策內容

**戰鬥規則不論對戰組合為何，皆為同一套規則。**

- **玩家 vs NPC**：與 **NPC vs NPC**、**玩家 vs 玩家** 使用相同判定邏輯、相同屬性與技能結算、相同戰鬥 log 語義。
- 不因「是否為玩家」而區分規則（無「PvE 專用」與「PvP 專用」兩套）；後端僅區分「攻擊方／防禦方」為哪一實體，不因實體是否為玩家而走不同流程。
- 與設計關鍵字一致：**戰鬥全自動、文字 log**；分秒輸贏靠**裝備、技能等數據（後端）屬性配置**（田忌賽馬概念）；**自創技能**需符合世界規則，且在同一套規則下對所有活體一體適用。

## 2. 語義與交互要點（對齊「後端唯一真理」與插頭／插座）

本專案採用**插頭／插座**語義（見 [決策 002：插頭／插座語義](002_sockets_plugs_semantic.md)）；戰鬥為其中一例：**Attack** 為插頭，目標需具備對應插座（`[Agent]`）。

以下要點與本專案相容，自參考文件擷取並內化；與本專案相左處（如他專案之街道節點、室內 layout、特定技術棧）已略過。

### 2.1 活體實體 (Agent) — 皆為戰鬥對象

- **定義**：玩家、NPC 等具備狀態與行為的實體；在本專案中**玩家＝NPC**，故語義上皆為 `[Agent]`。
- **戰鬥相關**：任一 `[Agent]` 對另一 `[Agent]` 的**攻擊 (Attack)**，皆走同一套戰鬥邏輯；目標具備「可被攻擊」之**邏輯插座**，後端依同一規則結算（HP、裝備、技能、屬性）。

### 2.2 動詞與目標匹配

- **Attack**：目標為 `[Agent]`（玩家或 NPC）。後端檢查目標是否為活體、是否可被攻擊；若可則執行**同一套**戰鬥流程（屬性、技能、裝備、log），不區分攻擊方／防禦方是否為玩家。
- 其餘動詞（如 Talk, Trade, Examine, Take, Drop 等）依本專案後續規格；本文件僅約束**戰鬥**為統一規則。

### 2.3 後端職責（戰鬥）

- **可執行性**：是否可攻擊、是否在範圍內、是否合法目標，皆由後端判斷；前端只發送「動作＋目標」，不預測結果。
- **戰鬥結算**：唯一真理在後端；裝備、技能、屬性、隨機因子等皆在後端計算，輸出為**文字 log** 與結果狀態；前端僅呈現後端回傳之敘事與數值（若可視化）。
- **交互行為**：距離、碰撞、能否攻擊等，後端為準；前端可做表現與插值，收到後端結果須校正。

### 2.4 前端邊界（與設計關鍵字一致）

- 前端不得自行判斷「可否攻擊」或預測勝負；僅依後端回傳之列表與結果顯示。
- 前端不得改動後端回傳的戰鬥敘事語義（誰贏誰輸、傷害、結果）；可做呈現優化，不可改語義。

## 3. 與設計關鍵字對齊

| 關鍵字 | 本決策對應 |
| --- | --- |
| 戰鬥全自動、文字 log | 戰鬥流程與結果由後端產出文字 log，前端僅顯示。 |
| 輸贏靠裝備／技能／後端屬性 | 同一套屬性與技能規則適用於所有對戰組合。 |
| 玩家＝NPC | 不因身份區分戰鬥規則；同一套規則涵蓋玩家 vs NPC、NPC vs NPC、玩家 vs 玩家。 |
| 後端唯一數據真理 | 可執行性、結算、log 皆在後端；前端為顯示窗口。 |
