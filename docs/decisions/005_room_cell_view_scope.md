# 決策 005：空間單位與視野範圍（房格一致）

**狀態**：已決斷  
**依據**：主管決斷

---

## 1. 決策內容

**一格 ＝ 一間房 ＝ 玩家視線所及的空間。**

三者為同一概念的不同表述：

| 用語 | 含義 |
| --- | --- |
| **格子** | 世界的最小空間單位（一個「格」）。 |
| **房間** | 一個 MUD 節點：有 id、名稱、描述、出口、房內實體（對應 `rooms`／`entity_room`）。 |
| **視線所及** | 玩家當前所在的那一個空間單位內能看到的一切。 |

- 玩家在「某一格」即為在「某一間房」，視線範圍即該房間；無半格或跨房視線。
- **視野內實體**＝與玩家**同一格／同一房**的實體（同房 NPC、其他玩家等）。

---

## 2. 與其他決策對齊

- **決策 004（視野內即時模擬）**：「視野內」依本決策為**與玩家同房**之 NPC；僅同房 NPC 進入即時模擬，其餘惰性＋事件回推。
- **房間制實作**：當前版本以房間為節點、出口相連；視野 API 回傳當前房間描述與同房實體，與本約定一致。
- 若日後擴充為 2D 俯視地圖：一格格子與房間可 1:1 對應，移動一格即沿出口進相鄰房。

---

## 3. 實作與規格

- 觀測觸發、坍縮範圍、視野內 NPC 列表：皆以**當前房間**為範圍。
- 規格與清單中「視野內」一詞，在房間制下均指**同房**；若有格點制擴充，一格對應一房時本約定仍適用。
